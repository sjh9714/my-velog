# [DAY2] [프로그래머스/Python] 정수 내림차순으로 배치하기 - 문자열 정렬과 join으로 풀기

## 문제 요약

정수 `n`이 주어진다.

`n`의 각 자릿수를 큰 수부터 작은 수 순서로 정렬한 뒤, 다시 정수로 반환하면 된다.

예를 들어 `n = 118372`라면 자릿수는 `1, 1, 8, 3, 7, 2`이다.
이를 내림차순으로 정렬하면 `8, 7, 3, 2, 1, 1`이 되고, 최종 정답은 `873211`이 된다.

## 처음 생각한 접근

처음에는 숫자의 각 자릿수를 어떻게 하나씩 다룰지부터 생각했다.

정수 상태에서는 자릿수를 바로 정렬하기 어렵다.
그래서 먼저 `str(n)`으로 정수를 문자열로 바꿨다.
문자열로 바꾸면 각 자릿수를 문자 하나씩 순회할 수 있다.

흐름은 다음과 같다.

1. 정수 `n`을 문자열로 바꾼다.
2. 각 문자를 리스트에 담는다.
3. 리스트를 내림차순으로 정렬한다.
4. 정렬된 문자를 다시 하나의 문자열로 이어 붙인다.
5. 최종 문자열을 정수로 변환해서 반환한다.

## 내가 작성한 코드

```python
def solution(n):
    answer = ""
    arr = []
    text = str(n)
    for i in text:
        arr.append(i)
    arr.sort(reverse=True)
    for i in arr:
        answer += i
    return int(answer)
```

## 코드 설명

먼저 정답을 만들 빈 문자열과 자릿수를 담을 빈 리스트를 준비한다.

```python
answer = ""
arr = []
```

그다음 정수 `n`을 문자열로 변환한다.

```python
text = str(n)
```

이제 문자열을 순회하면서 각 자릿수를 리스트에 담을 수 있다.

```python
for i in text:
    arr.append(i)
```

예를 들어 `n = 118372`라면 `arr`는 다음처럼 만들어진다.

```python
["1", "1", "8", "3", "7", "2"]
```

이 리스트를 내림차순으로 정렬한다.

```python
arr.sort(reverse=True)
```

`reverse=True`를 주면 큰 값부터 작은 값 순서로 정렬된다.

정렬된 문자들을 다시 하나의 문자열로 이어 붙인다.

```python
for i in arr:
    answer += i
```

마지막으로 문자열을 정수로 바꿔서 반환한다.

```python
return int(answer)
```

## 좋았던 점

처음 작성한 풀이는 자릿수를 다루는 과정을 단계별로 잘 나눴다.

- 정수를 문자열로 바꾼다.
- 문자열을 리스트로 만든다.
- 리스트를 내림차순으로 정렬한다.
- 다시 문자열로 합친다.
- 정수로 변환한다.

문제에서 요구하는 흐름이 코드에 그대로 드러나기 때문에, 처음 풀이로는 이해하기 쉽고 안정적인 방식이었다.

## 개선해볼 수 있는 코드

같은 흐름을 `sorted()`와 `join()`을 사용해서 한 줄로 줄일 수 있다.

```python
def solution_refactored(n):
    return int("".join(sorted(str(n), reverse=True)))
```

## 개선 코드 설명

먼저 `str(n)`으로 정수를 문자열로 바꾼다.

```python
str(n)
```

문자열은 순회 가능한 값이기 때문에 `sorted()`에 바로 넣을 수 있다.

```python
sorted(str(n), reverse=True)
```

이 코드는 각 자릿수를 내림차순으로 정렬한 문자 리스트를 반환한다.

예를 들어 `n = 118372`라면 결과는 다음과 같다.

```python
["8", "7", "3", "2", "1", "1"]
```

그다음 `join()`으로 문자 리스트를 하나의 문자열로 합친다.

```python
"".join(sorted(str(n), reverse=True))
```

반복문에서 문자열에 `+=`로 하나씩 붙일 수도 있지만, 여러 문자열을 합칠 때는 `join()`을 사용하는 편이 더 Python답고 효율적이다.

마지막으로 `int()`로 정수로 변환한다.

```python
int("".join(sorted(str(n), reverse=True)))
```

## 테스트 케이스

| n | expected | 설명 |
|---:|---:|---|
| `118372` | `873211` | 기본 예시 |
| `15` | `51` | 두 자리 수 |
| `1000000` | `1000000` | 0이 많은 경우 |
| `9876543210` | `9876543210` | 이미 내림차순인 경우 |
| `10200` | `21000` | 0이 중간에 섞인 경우 |
| `999` | `999` | 모든 자릿수가 같은 경우 |

## 시간복잡도

시간복잡도는 `O(d log d)`이다.

여기서 `d`는 정수 `n`의 자릿수다.
자릿수를 정렬하는 데 `O(d log d)`가 걸리기 때문이다.

처음 작성한 풀이도 정렬이 포함되어 있으므로 기본적으로 `O(d log d)`로 볼 수 있다.
다만 문자열을 반복해서 `+=`로 붙이는 부분은 입력이 커질수록 비효율적일 수 있다.

## 공간복잡도

공간복잡도는 `O(d)`이다.

정수를 문자열로 바꾸고, 각 자릿수를 담은 리스트 또는 정렬된 문자 리스트를 만들기 때문이다.

## 배운 점

- 정수의 각 자릿수를 다루려면 `str(n)`으로 문자열로 바꾸는 방법이 편하다.
- `arr.sort(reverse=True)`는 리스트를 제자리에서 내림차순 정렬한다.
- `sorted(str(n), reverse=True)`는 정렬된 문자 리스트를 반환한다.
- 여러 문자열을 합칠 때는 반복문에서 `+=`를 쓰기보다 `join()`을 떠올리자.
- 단계별 풀이로 먼저 정확하게 풀고, 이후 Python 내장 함수로 간결하게 개선할 수 있다.

## 최종적으로 기억할 코드

```python
def solution_refactored(n):
    return int("".join(sorted(str(n), reverse=True)))
```

최종적으로는 `str(n)`으로 자릿수를 문자로 다루고, `sorted(..., reverse=True)`와 `join()`으로 한 번에 합치는 흐름을 기억하면 좋겠다.

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- 정수의 각 자릿수를 다루는 문제는 숫자 계산보다 `str(n)` 변환이 더 단순할 때가 많다.
- 문자들을 정렬한 뒤 다시 문자열로 합칠 때는 `join()`을 떠올리자.
- 문자열 누적 `+=`보다 `''.join(...)`이 의도가 더 선명하고 Python답다.

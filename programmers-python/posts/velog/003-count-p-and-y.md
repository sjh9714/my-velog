# [DAY1] [프로그래머스/Python] 문자열 내 p와 y의 개수 - lower와 count로 풀기

## 문제 요약

문자열 `s`가 주어졌을 때, 문자열 안에 있는 `p`의 개수와 `y`의 개수가 같은지 확인하는 문제다.

대소문자는 구분하지 않는다.
예를 들어 `P`와 `p`는 같은 문자로 보고, `Y`와 `y`도 같은 문자로 본다.

또한 `p`와 `y`가 둘 다 하나도 없는 경우에는 개수가 모두 0이므로 `True`를 반환한다.

## 처음 생각한 접근

핵심은 대소문자를 어떻게 처리할지였다.

문자열을 그대로 두고 `p`, `P`, `y`, `Y`를 각각 세는 방법도 있지만, 그러면 비교해야 할 값이 많아진다.
그래서 먼저 문자열 전체를 소문자로 바꾼 뒤 `p`와 `y`만 비교하면 된다고 생각했다.

## 내가 작성한 코드

```python
def solution(s):
    text = s.lower()
    return text.count('p') == text.count('y')
```

## 코드 설명

먼저 `s.lower()`로 문자열 전체를 소문자로 바꾼다.

```python
s = "pPoooyY"
text = s.lower()  # "ppoooyy"
```

이렇게 바꿔두면 대문자인 `P`, `Y`를 따로 신경 쓰지 않아도 된다.

그다음 `count()`를 사용해서 `p`의 개수와 `y`의 개수를 비교한다.

```python
text.count('p') == text.count('y')
```

개수가 같으면 `True`, 다르면 `False`가 반환된다.

## 좋았던 점

이번 풀이는 문제 조건을 코드로 옮기는 과정이 깔끔했다.

- 대소문자를 무시한다 → `lower()`
- 특정 문자의 개수를 센다 → `count()`
- 두 개수가 같은지 확인한다 → `==`

이렇게 각 조건에 맞는 Python 문자열 메서드를 잘 사용하면 반복문 없이도 읽기 쉬운 풀이를 만들 수 있다.

## 개선해볼 수 있는 코드

현재 코드도 충분히 간단하지만, 같은 아이디어를 한 줄로 쓸 수도 있다.

```python
def solution_one_liner(s):
    return s.lower().count('p') == s.lower().count('y')
```

다만 이 방식은 `s.lower()`를 두 번 호출한다.
그래서 개인적으로는 처음 작성한 코드처럼 `text` 변수에 담아두는 편이 더 읽기 좋고, 불필요한 중복도 줄일 수 있어서 마음에 든다.

## 테스트 케이스

| s | expected | 설명 |
|---|----------|------|
| `"pPoooyY"` | `True` | p 2개, y 2개 |
| `"Pyy"` | `False` | p 1개, y 2개 |
| `"abc"` | `True` | p와 y가 모두 0개 |
| `"ppyy"` | `True` | p 2개, y 2개 |
| `"pyp"` | `False` | p 2개, y 1개 |
| `"PYpy"` | `True` | 대소문자 섞임 |

## 시간복잡도

시간복잡도는 `O(n)`이다.

`lower()`가 문자열 전체를 한 번 순회하고, `count()`도 문자열을 순회하기 때문이다.
상수 배의 차이는 있지만 입력 문자열 길이에 비례하므로 전체 시간복잡도는 `O(n)`으로 볼 수 있다.

## 공간복잡도

공간복잡도는 `O(n)`이다.

`s.lower()`가 소문자로 변환된 새 문자열을 만들기 때문이다.

## 배운 점

- 대소문자를 무시해야 할 때는 먼저 `lower()`나 `upper()`로 기준을 통일하면 편하다.
- 특정 문자의 개수를 비교할 때는 `count()`를 사용하면 코드가 짧고 직관적이다.
- 짧은 코드라도 중복 호출이 생긴다면, 중간 변수를 두는 편이 더 읽기 좋을 수 있다.

## 최종 코드

```python
def solution(s):
    text = s.lower()
    return text.count('p') == text.count('y')
```

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- 대소문자를 구분하지 않는 문자열 문제는 먼저 `lower()` 또는 `upper()`로 기준을 통일하자.
- 특정 문자의 개수 비교는 직접 반복문을 돌리기 전에 `count()`를 떠올리자.
- 조건식 자체가 True/False를 반환하면 바로 `return`할 수 있다.

## 마무리

이번 문제는 Python 문자열 메서드의 기본기를 확인하기 좋은 문제였다.

복잡한 알고리즘보다는 문제 조건을 정확히 읽고, 그 조건에 맞는 내장 메서드를 떠올리는 것이 중요했다.
특히 대소문자 처리처럼 작은 조건을 놓치면 정답이 달라질 수 있으니, 앞으로도 문제를 읽을 때 이런 디테일을 먼저 체크해야겠다.

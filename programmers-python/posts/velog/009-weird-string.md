# [DAY4] [프로그래머스/Python] 이상한 문자 만들기 - split이 아니라 공백을 보존해야 하는 문제

## 문제 요약

문자열 `s`가 주어진다.

각 단어에서 짝수 번째 문자는 대문자로, 홀수 번째 문자는 소문자로 바꾼 문자열을 반환하면 된다.
여기서 짝수/홀수는 전체 문자열 기준이 아니라 각 단어 안에서의 인덱스 기준이다.

문제 링크: [이상한 문자 만들기](https://school.programmers.co.kr/learn/courses/30/lessons/12930)

예를 들어 다음 문자열이 있다.

```python
s = "try hello world"
```

각 단어를 변환하면 다음과 같다.

```text
try   -> TrY
hello -> HeLlO
world -> WoRlD
```

그래서 정답은 `"TrY HeLlO WoRlD"`가 된다.

## 처음 작성한 코드

처음에는 문자열을 단어 단위로 나누면 쉽게 풀 수 있을 것 같았다.

```python
def solution(s):
    answer = ''
    arr = s.split()
    for i in arr:
        for j in range(len(i)):
            if j % 2:
                answer += i[j].lower()
            else:
                answer += i[j].upper()
        answer += ' '
    return answer.strip()
```

흐름은 다음과 같다.

1. `s.split()`으로 단어를 나눈다.
2. 각 단어에서 인덱스가 짝수면 대문자로 바꾼다.
3. 인덱스가 홀수면 소문자로 바꾼다.
4. 단어가 끝날 때마다 공백을 하나 붙인다.
5. 마지막 공백을 `strip()`으로 제거한다.

기본 예시처럼 단어 사이에 공백이 하나씩만 있는 경우에는 잘 동작했다.
하지만 제출해보니 정확성 테스트가 31.3점까지만 나왔다.

## 정확성 테스트 31.3점이 나온 이유

이 문제는 단어를 나누는 문제처럼 보이지만, 실제로는 원래 문자열의 공백까지 그대로 유지해야 하는 문제였다.

내 풀이의 핵심 문제는 `split()`이었다.

`split()`은 공백을 기준으로 단어를 나눌 때 편하지만, 원래 문자열에 있던 공백 정보를 그대로 보존하지 않는다.

예를 들어 다음 문자열을 보자.

```python
s = "hello  world"
```

단어 사이에 공백이 두 개 있다.
하지만 `split()`을 사용하면 결과는 이렇게 된다.

```python
["hello", "world"]
```

공백이 몇 개였는지 정보가 사라진다.

또 앞뒤 공백도 사라진다.

```python
"  hello".split()
```

결과는 다음과 같다.

```python
["hello"]
```

이처럼 `split()`은 연속된 공백과 앞뒤 공백 정보를 잃어버리기 때문에 일부 테스트에서 실패했다.

## split()이 놓친 부분

처음 풀이에서는 단어를 변환한 뒤 단어마다 공백을 하나씩 붙였다.

```python
answer += ' '
```

그리고 마지막에 `strip()`으로 양쪽 공백을 제거했다.

```python
return answer.strip()
```

이 방식은 원래 문자열의 공백이 항상 단어 사이에 하나씩만 있다고 가정한 풀이였다.

하지만 실제 테스트에는 이런 케이스가 들어올 수 있다.

```python
"  hello"
"hello  world"
"hello "
"  multiple   spaces  "
```

이런 입력에서는 공백 자체도 정답의 일부다.
따라서 공백을 새로 만들어 붙이면 안 되고, 원래 있던 공백을 그대로 결과에 넣어야 한다.

## 공백을 보존하는 접근

공백을 보존하려면 `split()`으로 단어를 먼저 나누기보다, 문자열을 한 글자씩 순회하는 편이 안전하다.

필요한 규칙은 단순하다.

1. 공백이 아닌 문자를 만나면 현재 단어 안에서의 인덱스를 기준으로 대소문자를 바꾼다.
2. 공백을 만나면 공백 자체를 결과에 넣는다.
3. 공백을 만나면 다음 단어가 시작되는 것이므로 단어 안에서의 인덱스를 0으로 초기화한다.

공백을 만나면 단어 안에서의 인덱스를 0으로 초기화하고, 공백 자체는 그대로 결과에 넣어야 한다.

## 개선한 코드

```python
def solution(s):
    answer = []
    index = 0

    for char in s:
        if char == " ":
            answer.append(char)
            index = 0
        else:
            answer.append(char.upper() if index % 2 == 0 else char.lower())
            index += 1

    return "".join(answer)
```

## 코드 설명

먼저 결과를 담을 리스트와 단어 안에서의 인덱스를 준비한다.

```python
answer = []
index = 0
```

문자열을 한 글자씩 순회한다.

```python
for char in s:
```

만약 현재 문자가 공백이라면, 공백을 그대로 결과에 추가한다.

```python
if char == " ":
    answer.append(char)
    index = 0
```

여기서 중요한 점은 `index = 0`이다.
공백 다음에는 새로운 단어가 시작되므로, 단어 안에서의 인덱스를 다시 0부터 세야 한다.

공백이 아니라면 현재 단어 안에서의 인덱스를 기준으로 대소문자를 바꾼다.

```python
answer.append(char.upper() if index % 2 == 0 else char.lower())
index += 1
```

짝수 번째 문자는 대문자, 홀수 번째 문자는 소문자로 바꾼 뒤 인덱스를 1 증가시킨다.

마지막으로 리스트에 담긴 문자를 하나의 문자열로 합친다.

```python
return "".join(answer)
```

문자열을 반복해서 `+=`로 붙이는 것보다 리스트에 담고 `join()`으로 합치는 방식이 더 안정적이다.

## 테스트 케이스

| s | expected | 설명 |
|---|---|---|
| `"try hello world"` | `"TrY HeLlO WoRlD"` | 기본 예시 |
| `"hello"` | `"HeLlO"` | 단어 하나 |
| `"a b c"` | `"A B C"` | 한 글자 단어 여러 개 |
| `"  hello"` | `"  HeLlO"` | 앞쪽 공백 보존 |
| `"hello  world"` | `"HeLlO  WoRlD"` | 연속 공백 보존 |
| `"hello "` | `"HeLlO "` | 뒤쪽 공백 보존 |
| `"  multiple   spaces  "` | `"  MuLtIpLe   SpAcEs  "` | 앞뒤/연속 공백 모두 보존 |

## 시간복잡도

시간복잡도는 `O(n)`이다.

문자열 `s`를 한 번 순회하기 때문이다.

## 공간복잡도

공간복잡도는 `O(n)`이다.

결과 문자열을 만들기 위해 문자들을 담는 리스트가 필요하다.

## 배운 점

- 문자열 문제에서 `split()`은 편하지만, 원래 공백을 보존해야 하는 문제에서는 위험할 수 있다.
- 정확성 테스트 일부만 통과했다면, 기본 예시에는 없는 공백/대소문자/경계값을 의심해보자.
- 단어 안에서의 인덱스와 전체 문자열 인덱스는 다를 수 있다.
- 공백을 만날 때마다 단어 인덱스를 초기화해야 한다.
- 문자열을 만들 때는 반복 `+=`보다 리스트에 담고 `join()`하는 방식을 떠올리자.

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- “단어별 인덱스”가 나오면 전체 문자열 인덱스가 아니라 단어 안의 인덱스를 따로 관리하자.
- 입력 문자열의 공백을 그대로 유지해야 하면 `split()`을 바로 쓰기 전에 한 번 멈추자.
- 연속 공백, 앞 공백, 뒤 공백 테스트를 직접 만들어보자.
- 공백이 구분자인 동시에 결과에 남아야 한다면 한 글자씩 순회하는 방식이 안전하다.

## 최종적으로 기억할 코드

```python
def solution(s):
    answer = []
    index = 0

    for char in s:
        if char == " ":
            answer.append(char)
            index = 0
        else:
            answer.append(char.upper() if index % 2 == 0 else char.lower())
            index += 1

    return "".join(answer)
```

## 마무리

이번 문제는 처음에는 단순한 문자열 변환 문제처럼 보였다.
하지만 실제 핵심은 대소문자 변환보다 공백 보존이었다.

내가 작성한 `split()` 기반 풀이는 기본 예시는 잘 통과했지만, 연속 공백과 앞뒤 공백을 잃어버려 정확성 테스트가 31.3점에서 멈췄다.

앞으로 문자열 문제를 풀 때는 `split()`을 쓰기 전에 “공백이 정답에 그대로 남아야 하는가?”를 먼저 확인해야겠다.

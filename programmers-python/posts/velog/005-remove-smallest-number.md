# [DAY2] [프로그래머스/Python] 제일 작은 수 제거하기 - min과 리스트 컴프리헨션으로 풀기

## 문제 요약

정수 배열 `arr`가 주어진다.

이 배열에서 가장 작은 수를 제거한 배열을 반환하면 된다.
단, 가장 작은 수를 제거한 뒤 배열이 비어 있다면 `[-1]`을 반환해야 한다.

예를 들어 `arr = [4, 3, 2, 1]`이라면 가장 작은 수는 `1`이므로 `[4, 3, 2]`를 반환한다.
`arr = [10]`처럼 원소가 하나뿐이라면 제거 후 빈 배열이 되므로 `[-1]`을 반환한다.

## 처음 생각한 접근

처음에는 가장 작은 수를 어떻게 찾을지에 집중했다.

가장 작은 수는 정렬했을 때 맨 앞에 오기 때문에, 먼저 `sorted(arr)`로 정렬된 배열을 만들었다.
그다음 정렬된 배열의 첫 번째 값을 최솟값으로 보고, 원본 배열에서 그 값을 찾아 제거하면 된다고 생각했다.

흐름은 다음과 같다.

1. `sorted(arr)`로 정렬된 배열을 만든다.
2. 정렬된 배열의 첫 번째 값을 최솟값으로 사용한다.
3. 원본 배열에서 최솟값의 인덱스를 찾는다.
4. `pop()`으로 해당 원소를 제거한다.
5. 배열이 비어 있으면 `[-1]`을 반환한다.

## 내가 작성한 코드

```python
def solution(arr):
    arr_c = sorted(arr)
    arr.pop(arr.index(arr_c[0]))
    return arr if arr else [-1]
```

## 코드 설명

먼저 `sorted(arr)`로 정렬된 새 리스트를 만든다.

```python
arr_c = sorted(arr)
```

`sorted()`는 원본 리스트를 직접 바꾸지 않고, 정렬된 새 리스트를 반환한다.
따라서 `arr_c[0]`은 배열에서 가장 작은 값이 된다.

```python
arr_c[0]
```

그다음 원본 배열에서 최솟값의 위치를 찾는다.

```python
arr.index(arr_c[0])
```

그리고 `pop()`으로 해당 위치의 원소를 제거한다.

```python
arr.pop(arr.index(arr_c[0]))
```

마지막으로 원소를 제거한 뒤에도 배열이 남아 있으면 `arr`를 그대로 반환하고, 비어 있다면 `[-1]`을 반환한다.

```python
return arr if arr else [-1]
```

Python에서 빈 리스트 `[]`는 조건식에서 `False`처럼 동작하기 때문에 이렇게 간단하게 처리할 수 있다.

## 좋았던 점

처음 작성한 풀이는 단계가 분명하다.

- 정렬해서 최솟값을 찾는다.
- 원본 배열에서 최솟값의 위치를 찾는다.
- 그 위치의 값을 제거한다.
- 빈 배열이면 `[-1]`을 반환한다.

문제의 요구사항을 그대로 코드로 옮긴 풀이이고, 정답도 잘 나온다.
처음 접근으로는 충분히 자연스러운 풀이였다.

## 개선해볼 수 있는 코드

다만 이 문제에서는 최솟값 하나만 필요하다.
전체 배열을 정렬하지 않아도 `min()`을 사용하면 최솟값을 바로 찾을 수 있다.

```python
def solution_refactored(arr):
    if len(arr) == 1:
        return [-1]

    min_value = min(arr)
    return [num for num in arr if num != min_value]
```

## 개선 코드 설명

먼저 배열의 길이가 1이면 어떤 값을 제거하더라도 빈 배열이 된다.
이 경우에는 바로 `[-1]`을 반환한다.

```python
if len(arr) == 1:
    return [-1]
```

그다음 `min(arr)`로 최솟값을 찾는다.

```python
min_value = min(arr)
```

`sorted(arr)`는 배열 전체를 정렬하므로 `O(n log n)`이 걸린다.
하지만 `min(arr)`는 배열을 한 번만 순회하므로 `O(n)`에 최솟값을 찾을 수 있다.

마지막으로 리스트 컴프리헨션을 사용해서 최솟값이 아닌 원소만 새 리스트에 담는다.

```python
[num for num in arr if num != min_value]
```

이 방식은 `pop()`처럼 원본 리스트를 직접 수정하지 않고, 결과 리스트를 새로 만든다는 점도 다르다.

## 테스트 케이스

| arr | expected | 설명 |
|---|---|---|
| `[4, 3, 2, 1]` | `[4, 3, 2]` | 가장 작은 값 1 제거 |
| `[10]` | `[-1]` | 제거 후 빈 배열 |
| `[1, 2, 3, 4]` | `[2, 3, 4]` | 맨 앞 최솟값 제거 |
| `[2, 1]` | `[2]` | 두 원소 중 최솟값 제거 |
| `[5, 3, 7, 2]` | `[5, 3, 7]` | 중간 이후에 있는 최솟값 제거 |

## 시간복잡도

처음 작성한 풀이는 `O(n log n)`이다.

`sorted(arr)`로 정렬하는 데 `O(n log n)`이 걸리고, `index()`로 최솟값의 위치를 찾는 데 `O(n)`이 걸린다.
전체적으로는 정렬 비용이 더 크기 때문에 `O(n log n)`으로 볼 수 있다.

개선 풀이의 시간복잡도는 `O(n)`이다.

`min(arr)`가 배열을 한 번 순회하고, 리스트 컴프리헨션도 배열을 한 번 순회한다.
따라서 전체 시간복잡도는 `O(n)`이다.

## 공간복잡도

처음 작성한 풀이는 `O(n)`이다.

`sorted(arr)`가 정렬된 새 리스트를 만들기 때문이다.

개선 풀이도 결과 리스트를 새로 만들기 때문에 공간복잡도는 `O(n)`이다.

## 배운 점

- 최솟값만 필요할 때는 전체 정렬보다 `min()`을 먼저 떠올리자.
- `sorted(arr)`는 정렬된 새 리스트를 반환한다.
- `pop()`은 원본 리스트를 직접 수정한다.
- 원본 리스트를 바꾸지 않고 결과를 만들고 싶다면 리스트 컴프리헨션을 사용할 수 있다.
- 정답은 나오지만, 필요한 작업이 “정렬”인지 “최솟값 찾기”인지 구분하면 더 효율적인 풀이를 만들 수 있다.

## 최종적으로 기억할 코드

```python
def solution_refactored(arr):
    if len(arr) == 1:
        return [-1]

    min_value = min(arr)
    return [num for num in arr if num != min_value]
```

최종적으로는 정렬 없이 `min()`으로 최솟값만 찾고, 리스트 컴프리헨션으로 최솟값을 제외한 새 리스트를 만드는 흐름을 기억하면 좋겠다.

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- 최솟값 하나만 필요하면 전체 정렬보다 `min()`을 먼저 떠올리자.
- 원본 리스트를 바꿔도 되는지 애매하면 새 리스트를 만드는 방식이 더 안전하다.
- 특정 값을 제외한 리스트가 필요하면 리스트 컴프리헨션으로 조건을 직접 표현할 수 있다.

## 마무리

처음 작성한 `sorted + index + pop` 풀이도 문제를 이해하기 쉬운 방식으로 잘 풀어낸 코드다.
다만 최솟값 하나만 필요할 때는 전체 정렬까지 하지 않고 `min()`을 쓰는 편이 더 효율적이라는 점을 배웠다.

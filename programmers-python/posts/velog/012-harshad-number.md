# [DAY5] [프로그래머스/Python] 하샤드 수 - 자릿수 합으로 나누어 떨어지는지 확인하기

## 문제 요약

양의 정수 `x`가 주어진다.

`x`의 자릿수 합을 구한 뒤, `x`가 그 합으로 나누어 떨어지는지 확인하면 된다.
나누어 떨어지면 하샤드 수이므로 `True`, 아니면 `False`를 반환한다.

문제 링크: [하샤드 수](https://school.programmers.co.kr/learn/courses/30/lessons/12947)

예를 들어 `x = 12`라면 자릿수 합은 `1 + 2 = 3`이다.
`12`는 `3`으로 나누어 떨어지므로 하샤드 수다.

## 처음 생각한 접근

이 문제의 핵심은 숫자의 각 자릿수를 어떻게 꺼낼 것인가였다.

정수를 그대로 두면 자릿수를 하나씩 다루기 어렵다.
그래서 먼저 `str(x)`로 문자열로 바꾸면 각 자릿수를 문자처럼 순회할 수 있다.

그다음 각 문자를 다시 `int()`로 바꿔 더하고, 원래 숫자 `x`가 그 합으로 나누어 떨어지는지 확인하면 된다.

## 내가 작성한 코드

```python
def solution(x):
    arr = list(str(x))
    s = 0
    for i in arr:
        s += int(i)
    return False if x % s else True
```

## 코드 설명

먼저 정수 `x`를 문자열로 바꾼 뒤 리스트로 만든다.

```python
arr = list(str(x))
```

예를 들어 `x = 18`이라면 다음과 같이 바뀐다.

```python
list(str(18))  # ["1", "8"]
```

자릿수 합을 저장할 변수 `s`를 0으로 시작한다.

```python
s = 0
```

각 자릿수 문자를 하나씩 꺼내서 정수로 바꾼 뒤 더한다.

```python
for i in arr:
    s += int(i)
```

마지막으로 `x`를 자릿수 합 `s`로 나누었을 때 나머지가 있는지 확인한다.

```python
return False if x % s else True
```

`x % s`가 0이면 나누어 떨어진다는 뜻이다.
Python에서 0은 조건식에서 `False`처럼 동작하므로, 나머지가 없을 때 `True`를 반환한다.

## 좋았던 점

처음 작성한 풀이는 자릿수를 다루기 위해 숫자를 문자열로 바꾸는 방향을 잘 잡았다.

자릿수 문제에서는 산술 연산으로 `//`, `%`를 반복해서 풀 수도 있지만, Python에서는 `str(x)`로 바꾸는 방식이 직관적이고 실수도 적다.
그리고 각 문자를 `int()`로 바꿔 더한 뒤 나머지를 확인하는 흐름도 문제 조건과 잘 맞았다.

## 개선해볼 수 있는 코드

같은 로직을 `sum()`과 `map()`을 사용해서 더 짧게 표현할 수 있다.

```python
def solution_refactored(x):
    digit_sum = sum(map(int, str(x)))
    return x % digit_sum == 0
```

## sum(map(int, str(x)))로 줄일 수 있는 이유

`str(x)`는 숫자의 각 자릿수를 문자로 만든다.

```python
str(123)  # "123"
```

`map(int, str(x))`는 각 문자를 정수로 바꾼다.

```python
map(int, "123")  # 1, 2, 3
```

마지막으로 `sum()`이 이 값들을 모두 더한다.

```python
sum(map(int, "123"))  # 6
```

즉, 아래 코드는 자릿수 합을 바로 구하는 표현이다.

```python
digit_sum = sum(map(int, str(x)))
```

그리고 `x % digit_sum == 0`처럼 직접 비교하면 반환값이 이미 `True` 또는 `False`가 되므로 삼항 연산자를 쓰지 않아도 된다.

## 테스트 케이스

| x | expected | 설명 |
|---:|---|---|
| `10` | `True` | `1 + 0 = 1`, 10은 1로 나누어 떨어짐 |
| `12` | `True` | `1 + 2 = 3`, 12는 3으로 나누어 떨어짐 |
| `11` | `False` | `1 + 1 = 2`, 11은 2로 나누어 떨어지지 않음 |
| `13` | `False` | `1 + 3 = 4`, 13은 4로 나누어 떨어지지 않음 |
| `18` | `True` | `1 + 8 = 9`, 18은 9로 나누어 떨어짐 |
| `100` | `True` | `1 + 0 + 0 = 1`, 100은 1로 나누어 떨어짐 |
| `101` | `False` | `1 + 0 + 1 = 2`, 101은 2로 나누어 떨어지지 않음 |

## 시간복잡도

시간복잡도는 `O(n)`이다.

여기서 `n`은 숫자 `x`의 자릿수다.
각 자릿수를 한 번씩 확인해서 합을 구하기 때문이다.

## 공간복잡도

공간복잡도는 `O(n)`이다.

`str(x)`로 자릿수만큼의 문자열을 만들기 때문이다.
사용자 풀이에서는 `list(str(x))`도 만들기 때문에 자릿수만큼의 추가 공간이 필요하다.

## 배운 점

- 정수의 자릿수를 다룰 때는 `str(x)`를 먼저 떠올릴 수 있다.
- 문자열로 바꾼 자릿수는 계산하기 전에 `int()`로 다시 변환해야 한다.
- 여러 숫자를 더할 때는 `sum()`을 사용할 수 있다.
- 반복 가능한 값 전체에 같은 변환을 적용할 때는 `map()`을 사용할 수 있다.
- 조건식이 이미 `True` 또는 `False`를 만든다면 그대로 반환해도 된다.

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- 숫자의 각 자릿수를 봐야 하면 `str(x)`로 바꿔서 순회할 수 있는지 먼저 생각하자.
- 자릿수 합을 구해야 하면 `sum(map(int, str(x)))`를 떠올리자.
- 나누어 떨어지는지 확인해야 하면 `%` 결과가 0인지 확인하자.
- `True if 조건 else False` 형태가 나오면 조건식 자체를 반환할 수 있는지 확인하자.

## 최종적으로 기억할 코드

```python
def solution_refactored(x):
    digit_sum = sum(map(int, str(x)))
    return x % digit_sum == 0
```

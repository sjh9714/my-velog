# [DAY1] [프로그래머스/Python] 두 정수 사이의 합 - 등차수열 합 공식으로 풀기

## 문제 요약

두 정수 `a`, `b`가 주어졌을 때, `a`와 `b` 사이에 있는 모든 정수의 합을 구하는 문제다.

예를 들어 `a = 3`, `b = 5`라면 `3 + 4 + 5 = 12`를 반환하면 된다.
두 수의 대소관계는 정해져 있지 않기 때문에 `a`가 더 클 때도 같은 방식으로 처리해야 한다.

## 처음 생각한 접근

처음에는 `a`와 `b` 사이에 있는 숫자의 개수를 직접적으로 생각했다.
두 수 사이의 거리인 `abs(a - b)`를 구한 뒤, 이 값이 홀수인지 짝수인지에 따라 합을 구하는 식을 다르게 세웠다.

이 접근도 정답은 나온다. 다만 경우를 나누다 보니 코드가 조금 길어지고, `/` 연산 때문에 마지막에 `int()`로 변환하는 과정도 필요했다.

## 내가 작성한 코드

```python
def solution(a, b):
    answer = 0
    if abs(a - b) % 2:
        answer = ((abs(a - b) + 1) / 2) * (a + b)
    else:
        answer = (abs(a - b) / 2) * (a + b) + ((a + b) / 2)
    return int(answer)
```

## 코드 설명

`abs(a - b)`는 두 수 사이의 거리를 의미한다.

- 거리가 홀수이면 양 끝 숫자를 묶었을 때 같은 합을 가지는 쌍이 만들어진다.
- 거리가 짝수이면 가운데 숫자가 하나 남기 때문에 그 값을 추가로 더해준다.

예를 들어 `3, 5`의 경우 `3 + 5`를 먼저 생각하고, 가운데 값인 `4`를 따로 더하는 방식이다.

## 아쉬웠던 점

정답은 맞지만, 문제 자체가 연속된 정수의 합을 구하는 문제라서 등차수열의 합 공식으로 더 간단하게 표현할 수 있었다.

처음에는 항의 개수가 홀수인지 짝수인지에 따라 경우를 나누었지만, 등차수열의 합 공식을 그대로 적용하면 조건문 없이 한 줄로 풀 수 있었다.

## 개선한 코드

```python
def solution_refactored(a, b):
    return (abs(a - b) + 1) * (a + b) // 2
```

## 개선 코드 설명

연속된 정수는 공차가 1인 등차수열이다.
따라서 다음 공식을 사용할 수 있다.

```text
합 = 항의 개수 * (첫 항 + 마지막 항) // 2
```

여기서 항의 개수는 `abs(a - b) + 1`이다.
예를 들어 `3`부터 `5`까지는 `3, 4, 5` 총 3개이므로 `abs(3 - 5) + 1 = 3`이 된다.

또한 Python에서 `/`는 float 나눗셈이고, `//`는 정수 나눗셈이다.
이 문제의 결과는 항상 정수이므로 `//`를 사용하면 마지막에 `int()`로 변환하지 않아도 된다.

## 테스트 케이스

| a | b | expected |
|---|---|----------|
| 3 | 5 | 12 |
| 3 | 3 | 3 |
| 5 | 3 | 12 |
| -3 | 1 | -5 |
| -5 | -3 | -12 |
| 0 | 0 | 0 |

## 시간복잡도

시간복잡도는 `O(1)`이다.
반복문을 사용하지 않고 공식 한 번으로 계산하기 때문이다.

## 배운 점

- 연속된 정수의 합은 등차수열의 합 공식으로 풀 수 있다.
- 조건문으로 경우를 나누기 전에 하나의 공식으로 통합할 수 있는지 먼저 생각해보면 좋다.
- Python에서 `/`와 `//`의 차이를 의식해야 한다.

## 최종 코드

```python
def solution(a, b):
    return (abs(a - b) + 1) * (a + b) // 2
```

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- 연속된 정수의 합이 나오면 하나씩 더하기 전에 등차수열 합 공식을 먼저 떠올리자.
- `a`와 `b`의 순서가 바뀔 수 있으면 `abs(a - b) + 1`로 항의 개수를 구할 수 있다.
- 정수 결과가 필요한 계산에서는 `/`와 `//`의 차이를 꼭 확인하자.

# [DAY11] [프로그래머스/Python] 가장 큰 수 - 문자열 정렬 key와 x * 3 이해하기

## 문제 요약

정수 배열 `numbers`가 주어진다.

배열의 숫자들을 이어 붙여 만들 수 있는 가장 큰 수를 문자열로 반환해야 한다.

문제 링크: [가장 큰 수](https://school.programmers.co.kr/learn/courses/30/lessons/42746)

예를 들어 아래 입력이 있다고 해보자.

```python
numbers = [6, 10, 2]
```

가장 큰 순서로 이어 붙이면 `6`, `2`, `10` 순서가 되고, 결과는 `"6210"`이다.

이 문제는 단순히 숫자를 큰 순서대로 정렬하는 문제가 아니다.
숫자를 문자열로 바꾼 뒤, 이어 붙였을 때 어떤 순서가 더 큰지를 비교해야 한다.

## 처음 생각한 접근

처음에는 숫자를 내림차순으로 정렬하면 될 것 같아 보인다.

하지만 바로 반례가 나온다.

```python
[3, 30]
```

숫자 크기만 보면 `30`이 `3`보다 크다.
그런데 이어 붙여 보면 다르다.

```text
3 + 30  -> "330"
30 + 3  -> "303"
```

`"330"`이 `"303"`보다 크기 때문에 `3`이 `30`보다 앞에 와야 한다.

그래서 이 문제는 숫자 자체의 크기가 아니라, 문자열을 이어 붙였을 때의 결과를 기준으로 정렬해야 한다.

## 내가 작성한 코드

```python
def solution(numbers):
    str_number = list(map(str, numbers))
    str_number.sort(key=lambda x: x * 3, reverse=True)
    answer = "".join(str_number)
    return "0" if answer[0] == "0" else answer
```

## 코드 설명

먼저 숫자 리스트를 문자열 리스트로 바꿨다.

```python
str_number = list(map(str, numbers))
```

이 문제의 최종 답도 문자열이고, 정렬 기준도 문자열 비교를 이용해야 하므로 숫자를 문자열로 바꾸는 것이 출발점이다.

그다음 정렬 기준으로 `lambda x: x * 3`을 사용했다.

```python
str_number.sort(key=lambda x: x * 3, reverse=True)
```

정렬이 끝나면 문자열을 모두 이어 붙인다.

```python
answer = "".join(str_number)
```

마지막으로 모든 숫자가 0인 경우를 처리한다.

```python
return "0" if answer[0] == "0" else answer
```

## 좋았던 점

문제를 숫자 정렬이 아니라 문자열 정렬로 바꿔서 생각한 점이 좋았다.

`map(str, numbers)`로 자료형을 먼저 바꾸고, `sort()`의 `key`에 비교 기준을 넣은 흐름도 명확하다.

특히 `lambda x: x * 3`은 처음 보면 낯설지만, 자리수가 다른 문자열을 비교하기 위한 핵심 장치였다.

## 왜 `x * 3`을 하는가

가장 중요한 부분은 이 코드다.

```python
str_number.sort(key=lambda x: x * 3, reverse=True)
```

문자열은 앞 글자부터 비교된다.
그런데 숫자의 자리수가 다르면 단순 문자열 정렬만으로는 이어 붙였을 때의 크기를 정확히 판단하기 어렵다.

예를 들어 `3`과 `30`을 보자.

```text
"3" * 3  -> "333"
"30" * 3 -> "303030"
```

문자열 비교에서는 `"333"`이 `"303030"`보다 크다.
그래서 `3`이 `30`보다 앞에 오게 된다.

실제로 이 순서로 이어 붙이면 아래처럼 더 큰 수가 된다.

```text
"3" + "30" = "330"
"30" + "3" = "303"
```

즉 `x * 3`은 숫자를 실제로 세 번 쓰겠다는 의미가 아니라, 정렬할 때 비교 기준을 충분히 길게 만들어주는 장치다.

프로그래머스 조건에서 `numbers`의 원소는 `0` 이상 `1000` 이하이다.
대부분의 숫자는 최대 3자리이고, `1000`은 네 자리지만 앞자리가 `1`이라 큰 수 조합에서 앞쪽으로 올라갈 가능성이 낮다.

그래서 이 문제의 대표적인 Python 풀이에서는 `x * 3` 패턴을 많이 사용한다.
핵심은 “서로 다른 자리수의 문자열을 반복해서 비교한다”는 점이다.

## 마지막 `0` 조건이 필요한 이유

이 문제에서 놓치기 쉬운 또 하나의 포인트는 모든 숫자가 0인 경우다.

```python
numbers = [0, 0, 0]
```

문자열로 바꾸고 정렬한 뒤 join하면 결과는 아래처럼 된다.

```python
answer = "000"
```

하지만 정답은 `"000"`이 아니라 `"0"`이다.

그래서 마지막에 첫 글자를 확인한다.

```python
return "0" if answer[0] == "0" else answer
```

정렬을 내림차순으로 했는데 첫 글자가 `"0"`이라는 것은 가장 큰 값도 0이라는 뜻이다.
즉 모든 숫자가 0이라는 의미다.

## `0` 조건을 안 쓰면 왜 위험한가

여기서 두 가지를 구분해야 한다.

단순히 아래처럼 작성하면 런타임 에러가 아니라 오답이 된다.

```python
return answer
```

`[0, 0, 0]`에서 `"000"`을 반환하기 때문이다.

그런데 이 문제를 해결하려고 아래처럼 `int()` 변환을 쓰는 경우가 있다.

```python
return str(int(answer))
```

이 방식은 작은 입력에서는 `"000"`을 `"0"`으로 바꿔주기 때문에 그럴듯해 보인다.
하지만 입력이 커지면 `answer`가 매우 긴 문자열이 될 수 있다.

긴 문자열을 정수로 변환하면 Python의 정수 문자열 변환 제한이나 변환 비용 때문에 런타임 에러가 날 수 있다.

그래서 이 문제에서는 `int(answer)`로 바꾸지 않고 문자열 상태에서 처리하는 편이 더 안전하다.

```python
return "0" if answer[0] == "0" else answer
```

이 조건문은 모든 0 케이스를 처리하면서도 거대한 문자열을 정수로 변환하지 않는다.

## 개선해볼 수 있는 코드

내가 작성한 코드는 충분히 통과 가능한 풀이이다.
다만 변수명을 조금 더 설명적으로 바꾸면 의도가 더 잘 보인다.

```python
def solution_refactored(numbers):
    number_strings = list(map(str, numbers))
    number_strings.sort(key=lambda number: number * 3, reverse=True)

    answer = "".join(number_strings)
    return "0" if answer[0] == "0" else answer
```

로직은 동일하다.
`number_strings`라는 이름을 사용하면 숫자를 문자열로 바꿔 정렬한다는 흐름이 더 잘 드러난다.

## 테스트 케이스

| numbers | expected | 설명 |
|---|---:|---|
| `[6, 10, 2]` | `"6210"` | 공식 예제 |
| `[3, 30, 34, 5, 9]` | `"9534330"` | 공식 예제 |
| `[0, 0, 0]` | `"0"` | 모든 값이 0인 경우 |
| `[0, 0, 1]` | `"100"` | 0과 양수가 섞인 경우 |
| `[12, 121]` | `"12121"` | 접두어처럼 겹치는 숫자 |
| `[1000, 0, 0]` | `"100000"` | 1000과 0이 섞인 경우 |

## 시간복잡도

시간복잡도는 `O(n log n)`이다.

문자열 변환은 `O(n)`이고, 핵심 비용은 정렬이다.
정렬은 `O(n log n)`이 걸린다.

정렬 key에서 `x * 3`을 만들지만 숫자 길이가 제한되어 있으므로 상수 비용으로 볼 수 있다.

## 공간복잡도

공간복잡도는 `O(n)`이다.

숫자를 문자열로 바꾼 리스트를 만들고, 마지막에 join한 결과 문자열을 저장한다.

## 배운 점

- 이어 붙여 가장 큰 수를 만드는 문제는 숫자 정렬이 아니라 문자열 정렬 문제로 봐야 한다.
- `3`과 `30`처럼 자리수가 다른 숫자는 실제로 이어 붙인 결과를 생각해야 한다.
- `lambda x: x * 3`은 자리수가 다른 문자열을 비교하기 위한 정렬 key다.
- 모든 숫자가 0이면 `"000"`이 아니라 `"0"`을 반환해야 한다.
- `str(int(answer))`로 처리하면 긴 문자열에서 런타임 에러 위험이 있으므로 문자열 상태에서 `answer[0] == "0"`으로 처리하는 편이 안전하다.

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- “숫자를 이어 붙여 가장 큰 수”가 나오면 숫자 크기 정렬을 의심하자.
- `a + b`와 `b + a` 중 어느 쪽이 큰지 생각하자.
- Python에서는 문자열 반복을 정렬 key로 사용할 수 있는지 확인하자.
- 모든 값이 0인 케이스를 마지막에 꼭 확인하자.
- 정답이 문자열인데 `int()`로 바꾸려는 순간, 입력 크기와 런타임 에러 가능성을 의심하자.

## 최종적으로 기억할 코드

```python
def solution(numbers):
    number_strings = list(map(str, numbers))
    number_strings.sort(key=lambda number: number * 3, reverse=True)

    answer = "".join(number_strings)
    return "0" if answer[0] == "0" else answer
```

## 마무리

가장 큰 수는 정렬 문제지만, 일반적인 숫자 정렬로는 풀 수 없는 문제였다.

핵심은 숫자를 문자열로 바꾸고, 이어 붙였을 때 더 큰 순서를 만들도록 정렬 key를 설계하는 것이다.

이번 문제에서 꼭 기억할 부분은 두 가지다.

`x * 3`으로 비교 기준을 만들고, 모든 숫자가 0인 경우는 `answer[0] == "0"`으로 문자열 상태에서 처리한다.

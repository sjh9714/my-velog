# [DAY5] [프로그래머스/Python] 콜라츠 추측 - 테스트 13번 실패와 num이 1인 경우

## 문제 요약

정수 `num`이 주어진다.

`num`이 1이 될 때까지 다음 작업을 반복한다.

1. 짝수라면 2로 나눈다.
2. 홀수라면 3을 곱하고 1을 더한다.
3. 500회 반복해도 1이 되지 않으면 `-1`을 반환한다.

문제 링크: [콜라츠 추측](https://school.programmers.co.kr/learn/courses/30/lessons/12943)

예를 들어 `num = 6`이라면 다음처럼 변한다.

```text
6 -> 3 -> 10 -> 5 -> 16 -> 8 -> 4 -> 2 -> 1
```

총 8번의 작업이 필요하므로 정답은 `8`이다.

## 처음 작성한 코드

```python
def solution(num):
    index = 0
    while index < 500:
        if num % 2:
            num = num * 3 + 1
        else:
            num /= 2
        index += 1
        if num == 1:
            return index
    return -1
```

대부분의 테스트는 통과했지만 테스트 13번만 실패했다.

처음에는 반복 횟수 조건이나 홀짝 처리에서 문제가 난 줄 알았다.
하지만 핵심 반례는 `num = 1`이었다.

## 테스트 13번만 실패한 이유

문제에서 이미 입력값이 1이라면 작업을 한 번도 하지 않아도 된다.

따라서 `num = 1`의 정답은 `0회`, 즉 `0`이다.

그런데 처음 작성한 코드는 반복문에 들어간 뒤에야 `num == 1`인지 확인한다.

```python
while index < 500:
    if num % 2:
        num = num * 3 + 1
```

`num = 1`은 홀수이므로 먼저 아래 연산이 실행된다.

```python
1 * 3 + 1  # 4
```

그래서 원래는 바로 `0`을 반환해야 하는데, 코드에서는 `1 -> 4 -> 2 -> 1` 과정을 거쳐 `3`을 반환한다.

이것이 테스트 13번 실패 원인이었다.

## num = 1 반례 확인

처음 작성한 코드를 `num = 1`로 실행하면 다음처럼 된다.

```python
solution_failed(1)  # 3
```

하지만 정답은 다음과 같아야 한다.

```python
solution(1)  # 0
```

이 문제는 반복 작업을 하기 전에 이미 종료 조건을 만족하는지 먼저 확인해야 했다.

## /= 대신 //=를 쓰는 이유

처음 코드에서는 짝수일 때 `/=`를 사용했다.

```python
num /= 2
```

Python에서 `/`는 float 나눗셈이다.
예를 들어 `4 / 2`의 결과는 `2.0`이다.

이 문제에서는 계속 정수를 다루는 것이 자연스럽기 때문에 정수 나눗셈인 `//`를 쓰는 편이 좋다.

```python
num //= 2
```

프로그래머스에서 이 코드가 바로 실패 원인이 되지는 않을 수 있지만, 정수 문제에서는 불필요하게 float로 바꾸지 않는 습관이 좋다.

## 개선한 코드

```python
def solution(num):
    count = 0

    while num != 1:
        if count == 500:
            return -1

        if num % 2:
            num = num * 3 + 1
        else:
            num //= 2

        count += 1

    return count
```

반복문 조건을 `while num != 1`로 바꿨다.

이렇게 하면 처음부터 `num = 1`인 경우에는 반복문에 들어가지 않고 바로 `count`인 0을 반환한다.

또 500회가 되면 `-1`을 반환하도록 반복문 안에서 횟수를 확인했다.

## for range(500)으로 정리한 코드

최대 반복 횟수가 500번으로 정해져 있으므로 `for`문으로도 표현할 수 있다.

```python
def solution_refactored(num):
    for count in range(500):
        if num == 1:
            return count

        if num % 2:
            num = num * 3 + 1
        else:
            num //= 2

    return -1
```

이 코드는 매 반복 시작 시점에 `num == 1`인지 먼저 확인한다.
그래서 `num = 1`이면 `count = 0`에서 바로 0을 반환한다.

500번 반복하는 동안 1이 되지 않으면 반복문이 끝나고 `-1`을 반환한다.

## 테스트 케이스

| num | expected | 설명 |
|---:|---:|---|
| `6` | `8` | 기본 예제 |
| `16` | `4` | 짝수만 이어지는 구간 포함 |
| `626331` | `-1` | 500회 안에 1이 되지 않는 경우 |
| `1` | `0` | 처음부터 1인 반례 |
| `2` | `1` | 한 번 나누면 1 |
| `3` | `7` | 홀수 규칙 확인 |

## 시간복잡도

시간복잡도는 `O(1)`이다.

최대 반복 횟수가 500회로 고정되어 있기 때문이다.
입력값이 커져도 문제 조건 안에서는 500번까지만 반복한다.

## 공간복잡도

공간복잡도는 `O(1)`이다.

추가로 사용하는 변수는 `count`와 갱신되는 `num` 정도다.

## 배운 점

- 반복문에 들어가기 전에 이미 종료 조건을 만족하는 입력이 있는지 확인해야 한다.
- `num = 1`처럼 가장 작은 입력값이 반례가 될 수 있다.
- Python에서 `/=`는 결과를 float로 만들 수 있고, `//=`는 정수 나눗셈이다.
- 최대 반복 횟수가 정해져 있으면 `for range(500)` 구조가 더 명확할 수 있다.
- 테스트 하나만 실패할 때는 경계값부터 먼저 확인하자.

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- 반복 작업 문제에서는 “처음부터 끝난 상태인가?”를 먼저 확인하자.
- 제한 횟수가 있으면 `while`보다 `for range(제한)`이 더 명확한지 생각하자.
- 정수 문제에서 나눗셈이 나오면 `/`와 `//` 중 무엇이 맞는지 확인하자.
- 특정 테스트만 실패하면 최소값, 최대값, 이미 정답인 상태를 반례로 넣어보자.

## 최종적으로 기억할 코드

```python
def solution_refactored(num):
    for count in range(500):
        if num == 1:
            return count

        if num % 2:
            num = num * 3 + 1
        else:
            num //= 2

    return -1
```

## 마무리

이번 문제는 로직 자체보다 종료 조건을 언제 확인하느냐가 중요했다.

처음 작성한 코드는 대부분의 경우에는 맞게 동작했지만, `num = 1`처럼 이미 목표 상태인 입력을 놓쳤다.
그래서 테스트 13번만 실패했고, 이 반례를 확인하자 원인을 명확히 찾을 수 있었다.

앞으로 반복문 문제를 풀 때는 반복을 시작하기 전에 이미 정답인 상태가 있는지 먼저 확인해야겠다.

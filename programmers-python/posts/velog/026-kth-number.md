# [DAY11] [프로그래머스/Python] K번째수 - slicing과 sorted로 구간 정렬하기

## 문제 요약

배열 `array`와 여러 개의 명령 `commands`가 주어진다.

각 명령은 `[i, j, k]` 형태다.
배열의 `i`번째부터 `j`번째까지 자르고, 그 부분 배열을 정렬했을 때 `k`번째 수를 구하면 된다.

문제 링크: [K번째수](https://school.programmers.co.kr/learn/courses/30/lessons/42748)

예를 들어 아래 입력이 있다고 해보자.

```python
array = [1, 5, 2, 6, 3, 7, 4]
commands = [[2, 5, 3], [4, 4, 1], [1, 7, 3]]
```

첫 번째 command `[2, 5, 3]`은 2번째부터 5번째까지 자른 뒤 정렬하고, 그중 3번째 값을 고르는 의미다.

```python
[5, 2, 6, 3] -> [2, 3, 5, 6] -> 5
```

따라서 전체 결과는 `[5, 6, 3]`이 된다.

## 처음 생각한 접근

처음에는 문제에서 시키는 순서를 그대로 코드로 옮기면 된다고 생각했다.

1. command를 하나씩 확인한다.
2. `array`에서 필요한 구간만 자른다.
3. 자른 배열을 정렬한다.
4. 정렬된 배열에서 `k`번째 값을 꺼낸다.
5. 결과 리스트에 담는다.

이 문제는 복잡한 알고리즘보다 Python의 slicing과 정렬을 정확히 쓰는지가 더 중요했다.

## 내가 작성한 코드

```python
def solution(array, commands):
    answer = []
    for i in commands:
        sliced_array = sorted(array[i[0] - 1:i[1]])
        answer.append(sliced_array[i[2] - 1])
    return answer
```

## 코드 설명

`commands`를 하나씩 순회하면서 각 command를 처리했다.

```python
for i in commands:
```

여기서 `i`는 `[시작 위치, 끝 위치, 고를 위치]` 형태의 리스트다.

문제의 위치 번호는 1부터 시작하지만, Python 리스트 인덱스는 0부터 시작한다.
그래서 시작 위치는 `i[0] - 1`로 바꿔야 한다.

```python
array[i[0] - 1:i[1]]
```

slicing에서 끝 인덱스는 포함되지 않는다.
그래서 `i[1]`을 그대로 쓰면 문제에서 말하는 `j`번째 원소까지 포함할 수 있다.

이후 `sorted()`로 자른 배열을 정렬한다.

```python
sliced_array = sorted(array[i[0] - 1:i[1]])
```

마지막으로 정렬된 배열에서 `k`번째 값을 꺼낸다.
여기도 Python 인덱스에 맞춰 `i[2] - 1`을 사용한다.

```python
answer.append(sliced_array[i[2] - 1])
```

## 좋았던 점

문제에서 요구하는 작업 순서를 그대로 코드로 옮겼기 때문에 흐름이 직관적이었다.

특히 slicing과 `sorted()`를 함께 사용해서 구간 정렬을 간단하게 처리했다.
Lv.1 정렬 문제에서는 이런 식으로 “필요한 구간만 잘라서 정렬한다”는 생각이 자주 쓰일 것 같다.

## slicing 인덱스에서 주의할 점

이 문제에서 가장 헷갈릴 수 있는 부분은 인덱스다.

문제에서는 `i번째`, `j번째`, `k번째`라고 말한다.
즉 1부터 세는 방식이다.

하지만 Python 리스트는 0부터 시작한다.

그래서 시작 위치와 k번째 값은 각각 `-1`을 해줘야 한다.

```python
array[start - 1:end]
sorted_array[k - 1]
```

다만 slicing의 끝 위치는 포함되지 않는다.
그래서 끝 위치는 `end - 1`이 아니라 `end`를 그대로 쓰면 된다.

이 부분이 이 문제의 핵심 포인트였다.

## 개선해볼 수 있는 코드

내가 작성한 코드는 충분히 잘 동작한다.
다만 command의 각 값이 무엇을 의미하는지 `i[0]`, `i[1]`, `i[2]`만 보면 바로 읽히지는 않는다.

구조분해를 사용하면 조금 더 의미가 분명해진다.

```python
def solution_refactored(array, commands):
    return [sorted(array[start - 1:end])[k - 1] for start, end, k in commands]
```

`start`, `end`, `k`라는 이름이 붙으니 command의 의미가 더 잘 보인다.

리스트 컴프리헨션을 사용해서 결과 리스트를 따로 만들고 `append()`하는 과정도 줄일 수 있다.

## 테스트 케이스

| array | commands | expected | 설명 |
|---|---|---:|---|
| `[1, 5, 2, 6, 3, 7, 4]` | `[[2, 5, 3], [4, 4, 1], [1, 7, 3]]` | `[5, 6, 3]` | 공식 예제 |
| `[3, 1, 2]` | `[[1, 3, 2]]` | `[2]` | command가 하나인 경우 |
| `[1, 2, 3, 4, 5]` | `[[1, 5, 5], [1, 5, 1]]` | `[5, 1]` | 전체 구간을 자르는 경우 |
| `[10, 20, 30]` | `[[2, 2, 1]]` | `[20]` | 한 칸 구간 |
| `[9, 8, 7, 6]` | `[[1, 4, 1], [1, 4, 4]]` | `[6, 9]` | 정렬이 필요한 경우 |

## 시간복잡도

각 command마다 구간을 자르고 정렬한다.

잘라낸 구간 길이를 `m`이라고 하면 정렬은 `O(m log m)`이다.
commands의 개수를 `c`, 배열 길이를 `n`이라고 하면 최악의 경우 `O(c * n log n)`으로 볼 수 있다.

## 공간복잡도

공간복잡도는 `O(m)`이다.

slicing으로 구간 리스트를 새로 만들고, 그 리스트를 정렬해서 사용하기 때문이다.
여기서 `m`은 command 하나에서 잘라낸 구간의 길이다.

## 배운 점

- 문제에서 주어지는 위치가 1부터 시작하는지 먼저 확인하자.
- Python slicing은 끝 인덱스를 포함하지 않는다.
- `array[start - 1:end]`처럼 시작점만 보정하면 `end`번째 원소까지 포함할 수 있다.
- `sorted()`는 원본 배열을 바꾸지 않고 정렬된 새 리스트를 반환한다.
- command 값이 여러 개라면 구조분해로 이름을 붙이면 읽기 쉬워진다.

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- “i번째부터 j번째까지 자른다”가 나오면 slicing을 먼저 떠올리자.
- “자른 뒤 정렬한다”가 나오면 `sorted(array[start:end])` 패턴을 생각하자.
- 문제의 번호가 1부터 시작하면 Python 인덱스로 바꿀 때 `-1`이 필요한지 확인하자.
- command가 `[i, j, k]`처럼 의미 있는 묶음이면 구조분해로 이름을 붙이자.

## 최종적으로 기억할 코드

```python
def solution(array, commands):
    return [sorted(array[start - 1:end])[k - 1] for start, end, k in commands]
```

## 마무리

K번째수는 정렬 문제라기보다 slicing 인덱스를 정확히 다루는 연습에 가까웠다.

풀이 자체는 짧지만, `start - 1`, `end`, `k - 1`이 각각 왜 필요한지 설명할 수 있어야 비슷한 문제에서 덜 헷갈릴 것 같다.

이번 문제에서 가져갈 핵심은 `array[start - 1:end]`와 `sorted_array[k - 1]`이다.

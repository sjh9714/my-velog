# [DAY2] [프로그래머스/Python] 나누어 떨어지는 숫자 배열 - 정렬과 조건 필터링으로 풀기

## 문제 요약

정수 배열 `arr`와 정수 `divisor`가 주어진다.

`arr`의 원소 중 `divisor`로 나누어 떨어지는 값만 골라서 오름차순으로 반환하면 된다.
만약 나누어 떨어지는 값이 하나도 없다면 `[-1]`을 반환한다.

예를 들어 `arr = [5, 9, 7, 10]`, `divisor = 5`라면 5로 나누어 떨어지는 값은 `5`와 `10`이므로 `[5, 10]`을 반환한다.

## 처음 생각한 접근

문제를 읽고 가장 먼저 떠올린 흐름은 단순했다.

1. 배열을 오름차순으로 정렬한다.
2. 배열을 하나씩 확인한다.
3. `divisor`로 나누어 떨어지는 값만 결과 배열에 넣는다.
4. 결과 배열이 비어 있으면 `[-1]`을 반환한다.

이 문제는 정답을 오름차순으로 반환해야 하므로, 먼저 정렬해두면 나중에 따로 정렬 결과를 신경 쓰지 않아도 된다.

## 내가 작성한 코드

```python
def solution(arr, divisor):
    answer = []
    arr.sort()
    for i in arr:
        if i % divisor == 0:
            answer.append(i)
    return answer if answer else [-1]
```

## 코드 설명

먼저 `answer`라는 빈 리스트를 만든다.

```python
answer = []
```

그다음 `arr.sort()`로 배열을 오름차순 정렬한다.

```python
arr.sort()
```

이렇게 하면 이후에 조건을 만족하는 숫자를 순서대로 추가하기만 해도 정답 배열이 오름차순이 된다.

배열을 순회하면서 `i % divisor == 0`인지 확인한다.

```python
for i in arr:
    if i % divisor == 0:
        answer.append(i)
```

`%` 연산자는 나머지를 구하는 연산자다.
어떤 수를 `divisor`로 나눴을 때 나머지가 0이면 나누어 떨어진다는 뜻이다.

마지막으로 `answer`가 비어 있지 않으면 그대로 반환하고, 비어 있다면 `[-1]`을 반환한다.

```python
return answer if answer else [-1]
```

Python에서 빈 리스트 `[]`는 조건식에서 `False`처럼 동작한다.
그래서 `answer if answer else [-1]`처럼 간단하게 처리할 수 있다.

## 좋았던 점

처음 작성한 풀이는 문제 요구사항과 코드 흐름이 거의 그대로 대응된다.

- 오름차순으로 반환한다 → `arr.sort()`
- 나누어 떨어지는 값만 고른다 → `if i % divisor == 0`
- 없으면 `[-1]`을 반환한다 → `answer if answer else [-1]`

정답도 잘 나오고, 단계별로 읽기 쉬운 풀이였다.
처음 문제를 풀 때는 이렇게 명확한 반복문으로 작성하는 방식이 실수를 줄이는 데 도움이 된다.

## 개선해볼 수 있는 코드

같은 아이디어를 리스트 컴프리헨션으로 조금 더 간결하게 쓸 수도 있다.

```python
def solution_refactored(arr, divisor):
    answer = [num for num in sorted(arr) if num % divisor == 0]
    return answer if answer else [-1]
```

## 개선 코드 설명

개선 코드에서는 `arr.sort()` 대신 `sorted(arr)`를 사용했다.

둘 다 정렬을 한다는 점은 같지만, 차이가 있다.

- `arr.sort()`는 원본 리스트 자체를 정렬한다.
- `sorted(arr)`는 원본 리스트는 그대로 두고, 정렬된 새 리스트를 반환한다.

그리고 리스트 컴프리헨션을 사용해서 조건을 만족하는 값만 바로 리스트로 만들었다.

```python
[num for num in sorted(arr) if num % divisor == 0]
```

이 코드는 정렬된 배열을 순회하면서 `num % divisor == 0`인 값만 `answer`에 담는다는 뜻이다.

처음 작성한 반복문 풀이도 충분히 좋지만, Python에서는 이런 조건 필터링을 리스트 컴프리헨션으로 표현하면 코드가 짧고 읽기 좋아질 때가 많다.

## 테스트 케이스

| arr | divisor | expected | 설명 |
|---|---:|---|---|
| `[5, 9, 7, 10]` | `5` | `[5, 10]` | 5와 10이 5로 나누어 떨어짐 |
| `[2, 36, 1, 3]` | `1` | `[1, 2, 3, 36]` | 모든 수가 1로 나누어 떨어지고 정렬 필요 |
| `[3, 2, 6]` | `10` | `[-1]` | 나누어 떨어지는 값이 없음 |
| `[1, 2, 3, 4]` | `2` | `[2, 4]` | 짝수만 필터링 |
| `[4, 8, 12]` | `4` | `[4, 8, 12]` | 이미 정렬된 배열 |
| `[7, 5, 9]` | `5` | `[5]` | 조건을 만족하는 값이 하나뿐 |

## 시간복잡도

시간복잡도는 `O(n log n)`이다.

배열을 정렬하는 데 `O(n log n)`이 걸린다.
그다음 배열을 한 번 순회하면서 조건을 확인하므로 `O(n)`이 추가된다.

전체적으로는 정렬 비용이 더 크기 때문에 `O(n log n)`으로 볼 수 있다.

## 공간복잡도

공간복잡도는 `O(n)`이다.

조건을 만족하는 값들을 담는 `answer` 리스트가 필요하기 때문이다.
개선 풀이에서는 `sorted(arr)`가 정렬된 새 리스트도 만들기 때문에 추가 공간을 사용한다.

## 배운 점

- 나누어 떨어지는 조건은 `num % divisor == 0`으로 확인할 수 있다.
- `arr.sort()`는 원본 리스트를 직접 바꾼다.
- `sorted(arr)`는 정렬된 새 리스트를 반환한다.
- 조건에 맞는 원소만 모을 때는 반복문과 `append()`로 먼저 풀고, 이후 리스트 컴프리헨션으로 개선해볼 수 있다.
- 정답을 오름차순으로 요구하는 문제에서는 정렬이 필요한지 먼저 확인하자.

## 최종 코드

```python
def solution(arr, divisor):
    answer = []
    arr.sort()
    for i in arr:
        if i % divisor == 0:
            answer.append(i)
    return answer if answer else [-1]
```

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- 조건에 맞는 원소만 모으는 문제는 필터링을 먼저 떠올리자.
- 결과를 오름차순으로 반환해야 하면 `sort()`와 `sorted()` 중 원본 변경 여부를 보고 고르자.
- 빈 리스트일 때 대체 값을 반환하는 패턴은 `answer if answer else [-1]`로 간단히 정리할 수 있다.

# [DAY5] [프로그래머스/Python] 약수의 합 - 나누어 떨어지는 수만 더하기

## 문제 요약

정수 `n`이 주어진다.

`n`의 약수를 모두 더한 값을 반환하면 된다.

문제 링크: [약수의 합](https://school.programmers.co.kr/learn/courses/30/lessons/12928)

예를 들어 `n = 12`라면 약수는 다음과 같다.

```text
1, 2, 3, 4, 6, 12
```

이 값들을 모두 더하면 `28`이므로 정답은 `28`이다.

## 처음 생각한 접근

약수는 어떤 수를 나누어 떨어뜨리는 수다.

즉 `i`가 `n`의 약수인지 확인하려면 `n % i == 0`인지 보면 된다.

그래서 1부터 `n`까지 전부 확인하면서, 나누어 떨어지는 값만 더하면 된다고 생각했다.

## 내가 작성한 코드

```python
def solution(n):
    answer = 0
    for i in range(1, n + 1):
        if n % i == 0:
            answer += i

    return answer
```

## 코드 설명

먼저 약수의 합을 저장할 변수 `answer`를 0으로 시작한다.

```python
answer = 0
```

그다음 1부터 `n`까지 순회한다.

```python
for i in range(1, n + 1):
```

여기서 `range(1, n + 1)`을 사용하는 이유는 `range()`의 끝 값은 포함되지 않기 때문이다.
따라서 `n`까지 확인하려면 끝 값을 `n + 1`로 써야 한다.

각 `i`가 `n`을 나누어 떨어뜨리는지 확인한다.

```python
if n % i == 0:
```

나누어 떨어진다면 `i`는 `n`의 약수이므로 정답에 더한다.

```python
answer += i
```

마지막으로 누적된 값을 반환한다.

```python
return answer
```

## 좋았던 점

처음 작성한 풀이는 약수의 정의를 그대로 코드로 옮겼다.

`n % i == 0`이라는 조건이 명확해서 읽기 쉽고, 왜 이 값이 더해지는지도 바로 이해할 수 있다.
Lv.1 수학 구현 문제에서는 이렇게 조건을 정확히 코드로 옮기는 것이 가장 중요하다고 느꼈다.

## 개선해볼 수 있는 코드

같은 로직을 `sum()`과 제너레이터 표현식으로 더 짧게 작성할 수 있다.

```python
def solution_refactored(n):
    return sum(i for i in range(1, n + 1) if n % i == 0)
```

## sum()과 제너레이터 표현식으로 줄일 수 있는 이유

이 문제에서 실제로 필요한 값은 조건을 만족하는 `i`들의 합이다.

즉 아래 반복문은

```python
answer = 0
for i in range(1, n + 1):
    if n % i == 0:
        answer += i
```

다음처럼 “약수만 골라서 더한다”로 표현할 수 있다.

```python
sum(i for i in range(1, n + 1) if n % i == 0)
```

여기서 `i for i in range(1, n + 1) if n % i == 0` 부분은 조건을 만족하는 `i`만 하나씩 만든다.

그리고 `sum()`이 그 값들을 모두 더한다.

## 테스트 케이스

| n | expected | 설명 |
|---:|---:|---|
| `12` | `28` | `1 + 2 + 3 + 4 + 6 + 12` |
| `5` | `6` | 소수라서 `1 + 5` |
| `1` | `1` | 1의 약수는 1 |
| `16` | `31` | 완전제곱수 포함 |
| `25` | `31` | `1 + 5 + 25` |
| `2` | `3` | 가장 작은 소수 |

## 시간복잡도

시간복잡도는 `O(n)`이다.

1부터 `n`까지 모든 수를 한 번씩 확인하기 때문이다.

## 공간복잡도

공간복잡도는 `O(1)`이다.

추가로 사용하는 변수는 누적합 `answer` 정도다.
개선 풀이의 제너레이터 표현식도 리스트를 따로 만들지 않고 값을 하나씩 넘긴다.

## 배운 점

- 약수 판별은 `n % i == 0`으로 할 수 있다.
- `range(1, n + 1)`은 1부터 n까지 확인할 때 자주 쓰인다.
- 조건에 맞는 값들을 더할 때는 `sum()`을 떠올릴 수 있다.
- 제너레이터 표현식은 리스트를 따로 만들지 않고 값을 하나씩 사용할 수 있다.
- 수학 구현 문제는 정의를 코드 조건으로 정확히 바꾸는 것이 중요하다.

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- “약수”, “나누어 떨어진다”가 나오면 `%` 연산자를 떠올리자.
- 1부터 n까지 모두 확인해야 하면 `range(1, n + 1)`을 먼저 생각하자.
- 조건에 맞는 값들의 합을 구한다면 `sum()`과 제너레이터 표현식으로 줄일 수 있는지 보자.
- 직접 누적하는 풀이를 먼저 만든 뒤, 한 줄 표현으로 바꿀 수 있는지 확인하자.

## 최종적으로 기억할 코드

```python
def solution_refactored(n):
    return sum(i for i in range(1, n + 1) if n % i == 0)
```

## 마무리

이번 문제는 약수의 정의를 코드로 옮기는 기본 구현 문제였다.

처음 작성한 코드는 `n % i == 0` 조건으로 약수를 찾고, 해당 값을 누적하는 방식이라 명확했다.
여기에 `sum()`과 제너레이터 표현식을 적용하면 같은 생각을 더 간결하게 표현할 수 있었다.

앞으로 약수나 배수 문제가 나오면 먼저 `%` 연산자로 나누어 떨어지는지 확인하는 습관을 가져야겠다.

# [DAY3] [프로그래머스/Python] 폰켓몬 - set과 min으로 고를 수 있는 종류 수 구하기

## 문제 요약

폰켓몬 종류 번호가 담긴 배열 `nums`가 주어진다.

전체 폰켓몬 중 절반인 `len(nums) // 2`마리를 선택할 수 있다.
이때 가장 많은 종류의 폰켓몬을 선택하면, 몇 종류까지 고를 수 있는지 반환하면 된다.

문제 링크: [폰켓몬](https://school.programmers.co.kr/learn/courses/30/lessons/1845)

예를 들어 다음과 같은 입력이 있다.

```python
nums = [3, 1, 2, 3]
```

전체 폰켓몬은 4마리이므로 2마리를 고를 수 있다.
종류는 `{1, 2, 3}`으로 총 3종류다.

하지만 2마리만 고를 수 있으므로 최대로 고를 수 있는 종류 수는 2다.

## 처음 생각한 접근

문제를 읽고 가장 먼저 떠올린 건 중복 제거였다.

폰켓몬 번호가 여러 번 나와도 같은 번호라면 같은 종류다.
그래서 서로 다른 종류가 몇 개인지 확인하려면 `set`을 사용하면 된다.

이 문제는 실제 폰켓몬을 어떤 순서로 고르는지가 아니라, 서로 다른 종류를 최대 몇 개까지 고를 수 있는지가 핵심이었다.

필요한 값은 두 가지다.

1. 고를 수 있는 마리 수: `len(nums) // 2`
2. 서로 다른 종류 수: `len(set(nums))`

정답은 `고를 수 있는 마리 수`와 `서로 다른 종류 수` 중 더 작은 값이다.

## 내가 작성한 코드

```python
def solution(nums):
    set_p = set(nums)
    return len(nums) // 2 if len(nums) / 2 < len(set_p) else len(set_p)
```

## 코드 설명

먼저 `set(nums)`로 중복을 제거한다.

```python
set_p = set(nums)
```

예를 들어 `nums = [3, 1, 2, 3]`이라면 `set_p`는 다음과 같다.

```python
{1, 2, 3}
```

이제 서로 다른 종류 수는 `len(set_p)`로 구할 수 있다.

```python
len(set_p)
```

고를 수 있는 마리 수는 전체 길이의 절반이다.

```python
len(nums) // 2
```

마지막으로 고를 수 있는 마리 수보다 종류 수가 더 많으면 고를 수 있는 마리 수를 반환하고, 그렇지 않으면 종류 수를 반환한다.

```python
return len(nums) // 2 if len(nums) / 2 < len(set_p) else len(set_p)
```

## 좋았던 점

처음 작성한 풀이는 문제의 핵심인 중복 제거를 잘 잡았다.

`set(nums)`를 사용하면 같은 종류의 폰켓몬을 하나로 묶을 수 있다.
그래서 복잡하게 조합을 만들거나 실제로 폰켓몬을 고르는 과정을 시뮬레이션하지 않아도 된다.

또한 선택 가능한 마리 수와 종류 수를 비교해야 한다는 점도 코드에 잘 드러난다.

## 개선해볼 수 있는 코드

조건문을 `min()`으로 더 짧게 표현할 수 있다.

```python
def solution_refactored(nums):
    return min(len(nums) // 2, len(set(nums)))
```

## min()으로 더 간단해지는 이유

이 문제의 정답은 결국 두 값 중 작은 값이다.

```python
고를 수 있는 마리 수 = len(nums) // 2
서로 다른 종류 수 = len(set(nums))
```

만약 종류 수가 아무리 많아도, 실제로 고를 수 있는 마리 수보다 더 많은 종류를 고를 수는 없다.
반대로 종류 수가 적다면, 선택 가능한 마리 수가 남아 있어도 그보다 많은 종류를 만들 수 없다.

그래서 정답은 아래처럼 정리된다.

```python
min(len(nums) // 2, len(set(nums)))
```

중복을 제거하고 종류 수만 확인할 때는 `set`을 먼저 떠올리면 된다.
그리고 두 값 중 작은 값을 고르는 문제라면 `min()`을 떠올리면 코드가 훨씬 단순해진다.

## 테스트 케이스

| nums | expected | 설명 |
|---|---:|---|
| `[3, 1, 2, 3]` | `2` | 2마리 선택 가능, 종류는 3개 |
| `[3, 3, 3, 2, 2, 4]` | `3` | 3마리 선택 가능, 종류도 3개 |
| `[3, 3, 3, 2, 2, 2]` | `2` | 3마리 선택 가능하지만 종류는 2개 |
| `[1, 2, 3, 4]` | `2` | 모든 종류가 다르지만 2마리만 선택 가능 |
| `[1, 1]` | `1` | 한 종류만 있음 |
| `[1, 2, 1, 2]` | `2` | 선택 가능 수와 종류 수가 같음 |

## 시간복잡도

시간복잡도는 `O(n)`이다.

`set(nums)`를 만들기 위해 `nums`를 한 번 순회한다.
그 외의 `len()`과 `min()`은 상수 시간으로 볼 수 있다.

## 공간복잡도

공간복잡도는 `O(n)`이다.

서로 다른 종류를 저장하는 `set`이 필요하기 때문이다.
최악의 경우 모든 폰켓몬이 서로 다른 종류라면 `set`에 `n`개의 값이 들어갈 수 있다.

## 배운 점

- 중복을 제거해야 할 때는 `set`을 먼저 떠올리자.
- 서로 다른 종류 수는 `len(set(nums))`로 쉽게 구할 수 있다.
- 고를 수 있는 개수와 실제 종류 수 중 작은 값을 고르는 문제는 `min()`으로 표현할 수 있다.
- `/`는 float 나눗셈이고 `//`는 정수 나눗셈이다.
- 문제를 실제 선택 과정으로 시뮬레이션하기 전에, 필요한 값만 계산해도 되는지 생각해보자.

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- “서로 다른 종류”, “중복 제거”, “몇 종류”라는 표현이 나오면 `set`을 떠올리자.
- 선택 가능한 개수에 제한이 있으면 `선택 가능 수`와 `후보 종류 수`를 비교하자.
- 두 값 중 작은 값이 정답이면 조건문 대신 `min()`으로 정리할 수 있다.

## 최종적으로 기억할 코드

```python
def solution_refactored(nums):
    return min(len(nums) // 2, len(set(nums)))
```

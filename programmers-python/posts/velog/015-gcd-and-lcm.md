# [DAY5] [프로그래머스/Python] 최대공약수와 최소공배수 - math.gcd와 math.lcm 사용하기

## 문제 요약

두 자연수 `n`, `m`이 주어진다.

두 수의 최대공약수와 최소공배수를 구한 뒤 배열에 담아 반환하면 된다.

문제 링크: [최대공약수와 최소공배수](https://school.programmers.co.kr/learn/courses/30/lessons/12940)

예를 들어 `n = 3`, `m = 12`라면 최대공약수는 `3`, 최소공배수는 `12`다.

```python
[3, 12]
```

## 처음 생각한 접근

이 문제는 최대공약수와 최소공배수를 구하는 문제다.

Python에서는 `math` 모듈에 이미 최대공약수를 구하는 `gcd()`와 최소공배수를 구하는 `lcm()`이 있다.

그래서 직접 유클리드 호제법을 구현하기보다, 표준 라이브러리를 사용해서 먼저 풀었다.

## 내가 작성한 코드

```python
import math


def solution(n, m):
    return [gcd(n, m), lcm(n, m)]


def gcd(n, m):
    return math.gcd(n, m)


def lcm(n, m):
    return math.lcm(n, m)
```

## 코드 설명

먼저 `math` 모듈을 가져온다.

```python
import math
```

`solution()`에서는 최대공약수와 최소공배수를 배열에 담아 반환한다.

```python
return [gcd(n, m), lcm(n, m)]
```

최대공약수는 `math.gcd()`로 구한다.

```python
def gcd(n, m):
    return math.gcd(n, m)
```

최소공배수는 `math.lcm()`으로 구한다.

```python
def lcm(n, m):
    return math.lcm(n, m)
```

## 좋았던 점

처음 작성한 풀이는 문제의 요구사항을 함수 이름으로 잘 분리했다.

`solution()` 안에서 바로 `math.gcd()`와 `math.lcm()`을 호출해도 되지만, `gcd()`, `lcm()` helper 함수로 나누니 “최대공약수를 구한다”, “최소공배수를 구한다”는 흐름이 잘 보였다.

또 표준 라이브러리를 활용해서 실수할 여지를 줄인 점도 좋았다.

## math.gcd()와 math.lcm()을 사용할 때 확인할 점

`math.gcd()`는 Python에서 오래 전부터 사용할 수 있는 함수다.

반면 `math.lcm()`은 Python 3.9부터 추가된 함수다.
현재 로컬 Python 환경에서는 사용할 수 있지만, 코딩테스트 환경에 따라 지원 여부를 확인하는 습관이 필요하다.

만약 `math.lcm()`을 사용할 수 없는 환경이라면 최소공배수 공식을 직접 사용하면 된다.

## 개선해볼 수 있는 코드

최소공배수는 두 수의 곱을 최대공약수로 나누어 구할 수 있다.

```python
def solution_refactored(n, m):
    gcd_value = math.gcd(n, m)
    lcm_value = n * m // gcd_value
    return [gcd_value, lcm_value]
```

이 코드는 `math.lcm()` 없이도 최소공배수를 구할 수 있다.
그래서 표준 라이브러리 지원 범위가 애매한 환경에서도 기억해두기 좋다.

## 최소공배수 공식 설명

두 수 `n`, `m`에 대해 아래 관계가 성립한다.

```python
n * m = gcd(n, m) * lcm(n, m)
```

따라서 최소공배수는 다음처럼 구할 수 있다.

```python
lcm = n * m // gcd
```

예를 들어 `n = 14`, `m = 21`이라면 최대공약수는 `7`이다.

```python
14 * 21 // 7  # 42
```

따라서 최소공배수는 `42`가 된다.

## 테스트 케이스

| n | m | expected | 설명 |
|---:|---:|---|---|
| `3` | `12` | `[3, 12]` | 기본 예제 |
| `2` | `5` | `[1, 10]` | 서로소 |
| `6` | `6` | `[6, 6]` | 두 수가 같은 경우 |
| `4` | `8` | `[4, 8]` | 한 수가 다른 수의 배수 |
| `9` | `27` | `[9, 27]` | 포함 관계 |
| `14` | `21` | `[7, 42]` | 공통 약수가 있는 경우 |

## 시간복잡도

시간복잡도는 `O(log min(n, m))`으로 볼 수 있다.

`math.gcd()`는 유클리드 호제법 기반으로 최대공약수를 구한다.
최소공배수 공식은 최대공약수를 구한 뒤 곱셈과 나눗셈만 하면 된다.

## 공간복잡도

공간복잡도는 `O(1)`이다.

결과 배열과 `gcd_value`, `lcm_value` 정도의 변수만 사용한다.

## 배운 점

- 최대공약수는 `math.gcd()`로 구할 수 있다.
- 최소공배수는 `math.lcm()`으로 구할 수 있다.
- `math.lcm()`을 사용할 수 없는 환경이라면 `n * m // gcd` 공식을 떠올리자.
- 최대공약수와 최소공배수는 `n * m = gcd * lcm` 관계로 연결된다.
- 코딩테스트에서 표준 라이브러리를 사용할 때는 지원 Python 버전을 확인하는 습관이 필요하다.

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- 최대공약수가 나오면 `math.gcd()`를 먼저 떠올리자.
- 최소공배수가 나오면 `math.lcm()` 또는 `n * m // gcd` 공식을 떠올리자.
- 두 수가 서로소인지 확인할 때는 최대공약수가 1인지 보면 된다.
- 표준 라이브러리 함수가 버전에 따라 다를 수 있으면 공식 풀이도 함께 기억하자.

## 최종적으로 기억할 코드

```python
import math


def solution_refactored(n, m):
    gcd_value = math.gcd(n, m)
    lcm_value = n * m // gcd_value
    return [gcd_value, lcm_value]
```

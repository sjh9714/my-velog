# [DAY10] [프로그래머스/Python] 폰켓몬 - set과 min으로 한 줄 정리하기

## 문제 요약

폰켓몬 종류 번호가 담긴 배열 `nums`가 주어진다.

전체 폰켓몬 중 절반인 `len(nums) // 2`마리를 고를 수 있을 때, 가장 많은 종류의 폰켓몬을 선택하면 몇 종류까지 고를 수 있는지 반환해야 한다.

문제 링크: [폰켓몬](https://school.programmers.co.kr/learn/courses/30/lessons/1845)

이 문제의 핵심은 실제로 어떤 폰켓몬을 고르는지보다, 서로 다른 종류를 최대 몇 개까지 고를 수 있는지 계산하는 것이다.

## Day3 때 풀었던 풀이 복습

Day3에 처음 풀었을 때는 먼저 중복을 제거한 뒤, 고를 수 있는 마리 수와 비교했다.

```python
def solution(nums):
    set_p = set(nums)
    return len(nums) // 2 if len(nums) / 2 < len(set_p) else len(set_p)
```

이 풀이도 정답은 나온다.

`set(nums)`로 서로 다른 종류 수를 구했고, 고를 수 있는 마리 수보다 종류 수가 많으면 `len(nums) // 2`를 반환했다.
반대로 종류 수가 더 적으면 모든 종류를 하나씩 고를 수 있으므로 `len(set_p)`를 반환했다.

다만 조건문으로 나누고 있다 보니, 문제의 핵심 공식이 조금 덜 바로 보였다.

## 이번에 작성한 한 줄 코드

이번에는 같은 생각을 `min()`으로 더 간단하게 정리했다.

```python
def solution(nums):
    return min(len(nums) // 2, len(set(nums)))
```

정답은 결국 아래 두 값 중 더 작은 값이다.

- 고를 수 있는 마리 수: `len(nums) // 2`
- 서로 다른 종류 수: `len(set(nums))`

고를 수 있는 마리 수보다 종류 수가 많아도, 실제로 고를 수 있는 마리 수를 넘을 수는 없다.
반대로 종류 수가 더 적으면, 아무리 많이 골라도 서로 다른 종류 수 이상으로 답이 커질 수 없다.

그래서 두 값 중 작은 값을 반환하면 된다.

## `set(nums)`의 의미

`set`은 중복을 제거하는 자료구조다.

예를 들어 아래 배열을 보자.

```python
nums = [3, 1, 2, 3]
```

여기서 `3`은 두 번 나오지만 같은 종류다.
따라서 `set(nums)`를 만들면 서로 다른 종류만 남는다.

```python
set(nums)  # {1, 2, 3}
```

이 문제에서는 각 종류가 몇 마리씩 있는지는 중요하지 않다.
서로 다른 종류가 몇 개인지만 알면 된다.

그래서 빈도 계산이 필요한 `Counter`나 `dict`보다 `set`이 더 단순하게 맞는다.

## `len(nums) // 2`의 의미

문제 조건에서 전체 폰켓몬 중 절반만 고를 수 있다.

```python
pick_count = len(nums) // 2
```

예를 들어 `nums`의 길이가 4라면 2마리를 고를 수 있고, 길이가 6이라면 3마리를 고를 수 있다.

아무리 폰켓몬 종류가 많아도 실제로 고를 수 있는 마리 수를 넘어서 선택할 수는 없다.
따라서 정답의 최대치는 항상 `len(nums) // 2`다.

## 왜 `min()`이면 충분한가

이 문제의 답은 두 제한 중 더 작은 값으로 결정된다.

```python
min(고를 수 있는 마리 수, 서로 다른 종류 수)
```

예를 들어 `[3, 1, 2, 3]`은 전체 4마리라서 2마리를 고를 수 있다.
서로 다른 종류는 `{1, 2, 3}`으로 3종류다.

```python
min(2, 3)  # 2
```

3종류가 있어도 2마리만 고를 수 있으므로 정답은 2다.

반대로 `[3, 3, 3, 2, 2, 2]`는 전체 6마리라서 3마리를 고를 수 있다.
하지만 서로 다른 종류는 `{2, 3}`으로 2종류뿐이다.

```python
min(3, 2)  # 2
```

3마리를 고를 수 있어도 종류는 2종류뿐이므로 정답은 2다.

## `Counter`나 `dict`가 필요 없는 이유

Day9에 다시 정리했던 `완주하지 못한 선수` 문제는 이름별 개수가 중요했다.
동명이인이 있을 수 있어서 `dict`나 `Counter`로 빈도를 세야 했다.

하지만 폰켓몬 문제는 다르다.

이 문제는 각 번호가 몇 번 나왔는지가 아니라, 서로 다른 번호가 몇 종류 있는지만 확인하면 된다.

그래서 아래처럼 개수를 세는 구조까지는 필요하지 않다.

```python
counts = {}
for num in nums:
    counts[num] = counts.get(num, 0) + 1
```

빈도 계산이 필요 없는 문제라면 `Counter`보다 `set`이 더 단순하다.

## 테스트 케이스

| nums | expected | 설명 |
|---|---:|---|
| `[3, 1, 2, 3]` | `2` | 종류는 3개지만 2마리만 선택 가능 |
| `[3, 3, 3, 2, 2, 4]` | `3` | 3마리 선택 가능, 종류도 3개 |
| `[3, 3, 3, 2, 2, 2]` | `2` | 3마리 선택 가능하지만 종류는 2개 |
| `[1, 2, 3, 4]` | `2` | 모두 다른 종류지만 절반만 선택 |
| `[1, 1]` | `1` | 한 종류만 존재 |
| `[1, 2, 1, 2]` | `2` | 선택 가능 수와 종류 수가 같음 |

## 시간복잡도

시간복잡도는 `O(n)`이다.

`set(nums)`를 만드는 과정에서 `nums`를 한 번 순회한다.
그 뒤 `len()`과 `min()`은 상수 시간으로 볼 수 있다.

## 공간복잡도

공간복잡도는 `O(n)`이다.

최악의 경우 모든 폰켓몬이 서로 다른 종류라면 `set`에 `n`개의 값이 들어갈 수 있다.

## 배운 점

- 서로 다른 종류 수만 필요하면 `set`을 먼저 떠올리자.
- 정답은 `고를 수 있는 마리 수`와 `서로 다른 종류 수` 중 작은 값이다.
- 조건문이 여러 갈래로 나뉘면 `min()`이나 `max()`로 표현할 수 있는지 생각해보자.
- 빈도 계산이 필요 없는 문제라면 `Counter`나 `dict`보다 `set`이 더 단순하다.

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- “서로 다른 종류”, “중복 제거”, “최대 몇 종류”가 보이면 `set`을 생각하자.
- 답이 두 제한 중 하나로 결정되면 `min()` 또는 `max()`로 정리할 수 있는지 보자.
- 각 값의 등장 횟수가 필요하면 `dict`나 `Counter`, 등장 여부나 종류 수만 필요하면 `set`을 선택하자.

## 최종적으로 기억할 코드

```python
def solution(nums):
    return min(len(nums) // 2, len(set(nums)))
```

## 마무리

폰켓몬은 해시 문제처럼 보이지만, 실제로는 빈도 계산까지 가지 않아도 되는 문제였다.

이번에는 Day3에 풀었던 풀이를 다시 보면서 조건문으로 나눴던 생각을 `min()` 한 줄로 정리할 수 있었다.

앞으로도 문제를 풀 때 “개수를 세야 하는지”와 “중복만 제거하면 되는지”를 먼저 구분해보면, 자료구조를 더 빠르게 고를 수 있을 것 같다.

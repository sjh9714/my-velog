# [DAY4] [프로그래머스/Python] 시저 암호 - ord와 chr로 알파벳 밀기

## 문제 요약

문자열 `s`와 자연수 `n`이 주어진다.

문자열의 각 알파벳을 `n`칸씩 밀어서 새로운 문자열을 반환하면 된다.
공백은 아무리 밀어도 공백 그대로 남아야 한다.

문제 링크: [시저 암호](https://school.programmers.co.kr/learn/courses/30/lessons/12926)

예를 들어 다음 입력이 있다.

```python
s = "AB"
n = 1
```

`A`를 한 칸 밀면 `B`, `B`를 한 칸 밀면 `C`가 되므로 정답은 `"BC"`다.

또 `z`를 한 칸 밀면 알파벳 끝을 넘어가므로 다시 `a`가 되어야 한다.

## 처음 생각한 접근

이 문제는 문자열을 한 글자씩 확인하면서 처리하면 된다.

다만 문자가 어떤 종류인지에 따라 처리 방식이 조금 다르다.

1. 공백이면 그대로 넣는다.
2. 대문자면 `A`를 기준으로 n칸 민다.
3. 소문자면 `a`를 기준으로 n칸 민다.

알파벳을 밀어야 하는 문제는 문자 자체보다 문자 코드의 위치를 숫자로 바꿔 생각하면 쉬워진다.

Python에서는 `ord()`와 `chr()`를 사용하면 문자와 숫자 코드를 서로 바꿀 수 있다.

## 내가 작성한 코드

```python
def solution(s, n):
    answer = []

    for char in s:
        if char == " ":
            answer.append(char)
        elif "A" <= char <= "Z":
            answer.append(chr((ord(char) - 65 + n) % 26 + 65))
        else:
            answer.append(chr((ord(char) - 97 + n) % 26 + 97))

    return "".join(answer)
```

## 코드 설명

먼저 변환된 문자를 담을 리스트를 만든다.

```python
answer = []
```

문자열을 한 글자씩 순회한다.

```python
for char in s:
```

공백이면 암호화하지 않고 그대로 결과에 넣는다.

```python
if char == " ":
    answer.append(char)
```

대문자라면 `A`를 기준으로 계산한다.

```python
elif "A" <= char <= "Z":
    answer.append(chr((ord(char) - 65 + n) % 26 + 65))
```

여기서 `65`는 대문자 `A`의 ASCII 코드다.

```python
ord("A")  # 65
```

`ord(char) - 65`를 하면 현재 문자가 `A`부터 몇 번째 위치에 있는지 알 수 있다.

예를 들어 `C`는 `A`에서 2칸 떨어져 있으므로 다음과 같이 볼 수 있다.

```python
ord("C") - 65  # 2
```

소문자는 `a`를 기준으로 계산한다.

```python
else:
    answer.append(chr((ord(char) - 97 + n) % 26 + 97))
```

`97`은 소문자 `a`의 ASCII 코드다.

마지막으로 리스트에 담긴 문자들을 하나의 문자열로 합친다.

```python
return "".join(answer)
```

## 좋았던 점

처음 작성한 풀이는 대문자, 소문자, 공백을 명확히 나눠서 처리했다.

특히 공백은 암호화 대상이 아니라는 점을 먼저 분리한 것이 좋았다.
그리고 대문자와 소문자의 기준점이 다르다는 점도 조건문으로 잘 나눴다.

문자를 바로 다루기보다 `ord()`로 숫자 위치로 바꾼 뒤 다시 `chr()`로 문자로 되돌리는 흐름이 이 문제와 잘 맞았다.

## % 26이 필요한 이유

알파벳은 총 26개다.

`z` 다음은 `a`, `Z` 다음은 `A`로 돌아와야 하므로 `% 26`으로 알파벳 범위 안에서 순환시켰다.

예를 들어 `z`를 1칸 민다고 생각해보자.

```python
ord("z") - ord("a")  # 25
```

`z`는 소문자 기준 25번째 위치다.
여기서 1을 더하면 26이 된다.

하지만 알파벳 위치는 0부터 25까지만 있으므로 다시 0으로 돌아와야 한다.

```python
(25 + 1) % 26  # 0
```

그래서 다시 `a`가 된다.

대문자도 같은 방식이다.
`Z` 다음은 `A`로 돌아와야 하므로 대문자도 `% 26`이 필요하다.

## 개선해볼 수 있는 코드

같은 로직을 helper 함수로 분리하면 기준 문자를 더 명확하게 표현할 수 있다.

```python
def solution_refactored(s, n):
    def shift(char, base):
        return chr((ord(char) - ord(base) + n) % 26 + ord(base))

    answer = []

    for char in s:
        if char == " ":
            answer.append(char)
        elif char.isupper():
            answer.append(shift(char, "A"))
        else:
            answer.append(shift(char, "a"))

    return "".join(answer)
```

이렇게 쓰면 `65`, `97` 같은 숫자를 직접 외우지 않아도 된다.
대문자는 `"A"`를 기준으로, 소문자는 `"a"`를 기준으로 민다는 의도가 더 잘 보인다.

대문자와 소문자는 ASCII 기준점이 다르기 때문에 각각 `A`, `a`를 기준으로 계산해야 한다.

## 테스트 케이스

| s | n | expected | 설명 |
|---|---:|---|---|
| `"AB"` | `1` | `"BC"` | 기본 대문자 |
| `"z"` | `1` | `"a"` | 소문자 순환 |
| `"a B z"` | `4` | `"e F d"` | 공백, 대문자, 소문자 혼합 |
| `"Z"` | `1` | `"A"` | 대문자 순환 |
| `"Hello World"` | `5` | `"Mjqqt Btwqi"` | 문장 형태 |
| `"A a Z z"` | `25` | `"Z z Y y"` | 25칸 이동 |
| `"   "` | `3` | `"   "` | 공백만 있는 문자열 |

## 시간복잡도

시간복잡도는 `O(n)`이다.

문자열 `s`를 한 번 순회하면서 각 문자를 변환하기 때문이다.

## 공간복잡도

공간복잡도는 `O(n)`이다.

변환된 문자를 담는 `answer` 리스트가 필요하고, 마지막에 문자열로 합친다.

## 배운 점

- 문자를 숫자 코드로 바꿀 때는 `ord()`를 사용한다.
- 숫자 코드를 다시 문자로 바꿀 때는 `chr()`를 사용한다.
- 알파벳이 끝에서 처음으로 돌아와야 하면 `% 26`을 떠올리자.
- 대문자와 소문자는 기준점이 다르므로 각각 `A`, `a`를 기준으로 계산해야 한다.
- ASCII 코드 숫자를 직접 쓰는 것보다 `ord("A")`, `ord("a")`처럼 표현하면 의도가 더 선명해진다.

## 다음에 비슷한 문제를 만나면 떠올릴 신호

- 알파벳을 몇 칸 이동시키는 문제가 나오면 `ord()`와 `chr()`를 떠올리자.
- 원형으로 다시 돌아와야 하는 구조라면 `%` 연산자를 떠올리자.
- 대문자와 소문자가 섞여 있으면 기준점을 따로 잡아야 하는지 확인하자.
- 공백이나 특수문자가 변환 대상인지 아닌지 먼저 분리하자.

## 최종적으로 기억할 코드

```python
def solution_refactored(s, n):
    def shift(char, base):
        return chr((ord(char) - ord(base) + n) % 26 + ord(base))

    answer = []

    for char in s:
        if char == " ":
            answer.append(char)
        elif char.isupper():
            answer.append(shift(char, "A"))
        else:
            answer.append(shift(char, "a"))

    return "".join(answer)
```

이번 문제는 Day4 문자열 변환 문제 중에서도 문자와 숫자 코드 사이를 오가는 연습이 되는 문제였다.

처음 작성한 코드는 공백, 대문자, 소문자를 분리해서 안정적으로 처리했다.
여기에 `ord("A")`, `ord("a")`를 기준으로 삼는 helper 함수를 붙이면 숫자 상수를 외우지 않고도 더 읽기 좋은 코드로 정리할 수 있었다.

앞으로 알파벳 순환 문제가 나오면 `현재 위치 -> 이동 -> % 26 -> 기준점 더하기` 흐름을 먼저 떠올리면 좋겠다.

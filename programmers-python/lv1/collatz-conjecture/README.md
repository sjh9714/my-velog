# 콜라츠 추측

## 문제 요약

정수 `num`이 주어진다.

아래 작업을 반복해서 `num`이 1이 되는 데 필요한 횟수를 반환한다.

1. 입력된 수가 짝수라면 2로 나눈다.
2. 입력된 수가 홀수라면 3을 곱하고 1을 더한다.
3. 500회 반복해도 1이 되지 않으면 -1을 반환한다.

문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/12943

## 접근 방식

반복 횟수를 세면서 `num`이 1이 될 때까지 규칙을 적용한다.

주의할 점은 처음부터 `num == 1`인 경우다.
이때는 아무 작업도 하지 않았으므로 정답이 0회, 즉 0이어야 한다.

## 테스트 13번 실패 원인

처음 작성한 코드는 반복문 안에서만 `num == 1`을 확인했다.
그래서 입력이 1이면 바로 0을 반환하지 못하고, 1을 홀수로 처리해서 `1 * 3 + 1 = 4`로 바꿔버린다.

즉 `num = 1` 반례에서 기존 코드는 3을 반환하지만, 정답은 0이다.
또 기존 코드는 짝수 처리에서 `/=`를 사용했지만, 정수 문제에서는 float 나눗셈보다 `//=` 정수 나눗셈을 쓰는 편이 안전하다.

## 핵심 코드

```python
def solution(num):
    count = 0

    while num != 1:
        if count == 500:
            return -1

        if num % 2:
            num = num * 3 + 1
        else:
            num //= 2

        count += 1

    return count
```

## 개선한 코드

```python
def solution_refactored(num):
    for count in range(500):
        if num == 1:
            return count

        if num % 2:
            num = num * 3 + 1
        else:
            num //= 2

    return -1
```

## 시간복잡도

O(1)

최대 반복 횟수가 500회로 고정되어 있다.

## 공간복잡도

O(1)

추가로 사용하는 변수는 `count`, `num` 정도다.

## 배운 점

- 반복 작업을 시작하기 전에 이미 종료 조건을 만족하는지 먼저 확인해야 한다.
- Python에서 `/`는 float 나눗셈이고, `//`는 정수 나눗셈이다.
- 최대 반복 횟수가 정해져 있으면 `for range(횟수)`로 표현할 수 있다.
- `num = 1`처럼 가장 작은 입력값이 반례가 되는지 꼭 확인하자.

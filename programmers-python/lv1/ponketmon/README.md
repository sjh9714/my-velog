# 폰켓몬

## 문제 요약

폰켓몬 종류 번호가 담긴 배열 `nums`가 주어진다.
전체 폰켓몬 중 절반인 `len(nums) // 2`마리를 고를 수 있을 때, 가장 많은 종류의 폰켓몬을 선택하면 몇 종류를 고를 수 있는지 반환한다.

문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/1845

## 접근 방식

이 문제는 실제 폰켓몬을 어떤 순서로 고르는지가 아니라, 서로 다른 종류를 최대 몇 개까지 고를 수 있는지가 핵심이다.

중복된 종류는 한 종류로만 세면 되므로 `set(nums)`로 중복을 제거했다.
정답은 `고를 수 있는 마리 수`와 `서로 다른 종류 수` 중 더 작은 값이다.

## 핵심 코드

```python
def solution_refactored(nums):
    return min(len(nums) // 2, len(set(nums)))
```

## 시간복잡도

O(n)

`set(nums)`를 만들기 위해 배열을 한 번 순회한다.

## 공간복잡도

O(n)

서로 다른 종류를 저장하는 `set`이 필요하다.

## 배운 점

- 중복을 제거하고 종류 수만 확인할 때는 `set`을 먼저 떠올리면 된다.
- 최대 선택 개수와 실제 종류 수 중 작은 값을 고르는 문제는 `min()`으로 간단하게 표현할 수 있다.
- `/`는 float 나눗셈이고, `//`는 정수 나눗셈이다.

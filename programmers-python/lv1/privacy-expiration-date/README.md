# 개인정보 수집 유효기간

## 문제 요약

오늘 날짜 `today`, 약관별 유효기간 `terms`, 개인정보 수집 정보 `privacies`가 주어진다.
각 개인정보가 오늘 기준으로 파기 대상인지 확인하고, 파기해야 하는 개인정보 번호를 반환한다.

문제 링크: [개인정보 수집 유효기간](https://school.programmers.co.kr/learn/courses/30/lessons/150370)

## 접근 방식

문제에서는 모든 달이 28일까지 있다고 가정한다.
따라서 날짜를 직접 비교하기보다, 날짜를 하나의 일수로 변환하면 계산이 단순해진다.

```python
to_days(date) = year * 12 * 28 + month * 28 + day
```

수집일을 일수로 바꾸고 약관 기간을 `개월 * 28`일로 더한 값이 오늘보다 작거나 같으면 파기 대상이다.

## 핵심 아이디어

- 약관 종류별 기간은 `terms_dict`에 저장한다.
- 날짜는 28일 기준 일수로 변환한다.
- `수집일 + 유효기간 * 28 <= 오늘`이면 이미 유효기간이 끝났으므로 파기 대상이다.
- 코드에서는 이 기준을 `expiration_days <= today_days`처럼 비교한다.
- 결과 번호는 1번부터 시작하므로 `enumerate(privacies, start=1)`를 쓰면 좋다.

## 시간복잡도

`O(t + p)`

`t`는 약관 수, `p`는 개인정보 수다.
약관 목록과 개인정보 목록을 각각 한 번씩 순회한다.

## 공간복잡도

`O(t + p)`

약관 종류별 기간을 저장하는 딕셔너리와 파기 대상 번호를 담는 결과 리스트를 사용한다.

## 배운 점

- 날짜 비교 문제는 기준 단위로 변환하면 훨씬 단순해질 수 있다.
- 문제에서 모든 달이 28일이라고 했으므로 실제 달력 계산을 할 필요가 없다.
- 번호가 1부터 시작하는 순회에는 `enumerate(..., start=1)`가 잘 맞는다.

# K번째수

## 문제 요약

배열 `array`와 여러 개의 명령 `commands`가 주어진다.
각 명령은 `[i, j, k]` 형태이며, 배열의 `i`번째부터 `j`번째까지 자른 뒤 정렬했을 때 `k`번째 수를 구한다.

문제 링크: [K번째수](https://school.programmers.co.kr/learn/courses/30/lessons/42748)

## 접근 방식

각 command마다 필요한 구간만 slicing으로 잘라낸다.
자른 배열을 `sorted()`로 정렬한 뒤, `k - 1` 인덱스의 값을 결과 리스트에 추가한다.

## 핵심 아이디어

- 문제의 `i`, `j`, `k`는 1부터 시작하는 번호다.
- Python 리스트 인덱스는 0부터 시작하므로 시작점은 `i - 1`로 바꾼다.
- slicing의 끝 인덱스는 포함되지 않으므로 `array[i - 1:j]`로 쓰면 `j`번째 원소까지 포함된다.
- 정렬한 배열에서 `k`번째 값은 `sorted_array[k - 1]`이다.

## 시간복잡도

각 command마다 구간 길이를 `m`이라고 하면 정렬에 `O(m log m)`이 걸린다.
전체 command 수를 `c`라고 하면 최악의 경우 `O(c * n log n)`으로 볼 수 있다.

## 공간복잡도

구간을 잘라낸 새 리스트와 정렬 결과를 사용하므로 command 하나를 처리할 때 `O(m)`의 추가 공간을 사용한다.

## 배운 점

Python slicing은 끝 인덱스를 포함하지 않는다.
문제에서 주어진 순서가 1부터 시작하는지, Python 인덱스처럼 0부터 시작하는지 먼저 확인해야 한다.
정렬 후 몇 번째 값을 고르는 문제에서는 `k - 1` 변환을 놓치지 말자.

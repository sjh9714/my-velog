# 키패드 누르기

## 문제 요약

숫자 배열 `numbers`와 주 손잡이 `hand`가 주어진다.
각 숫자를 누를 때 어떤 손가락을 사용했는지 `L`, `R` 문자열로 반환한다.

왼쪽 열인 `1`, `4`, `7`은 왼손으로 누른다.
오른쪽 열인 `3`, `6`, `9`는 오른손으로 누른다.
가운데 열인 `2`, `5`, `8`, `0`은 현재 양손 위치에서 더 가까운 손으로 누르고, 거리가 같으면 주 손잡이를 사용한다.

## 접근 방식

키패드를 2차원 좌표로 바꾸면 각 손가락과 목표 숫자 사이의 거리를 계산하기 쉽다.

손가락은 상하좌우로만 움직일 수 있으므로, 거리는 행 차이와 열 차이의 합인 맨해튼 거리로 계산한다.

## 핵심 아이디어

```python
left_dist = abs(left_pos[0] - target[0]) + abs(left_pos[1] - target[1])
right_dist = abs(right_pos[0] - target[0]) + abs(right_pos[1] - target[1])
```

가운데 열 숫자를 누른 뒤에는 사용한 손가락의 위치를 반드시 목표 숫자 위치로 갱신해야 한다.

## 시간복잡도

`O(n)`

`n`은 `numbers`의 길이다.
각 숫자마다 상수 시간 안에 위치와 거리를 계산한다.

## 공간복잡도

`O(n)`

정답 문자열을 저장하는 공간이 필요하다.
키패드 좌표 딕셔너리는 숫자 개수가 고정되어 있으므로 `O(1)`로 볼 수 있다.

## 배운 점

- 키패드나 격자 문제는 좌표로 바꾸면 단순해진다.
- 상하좌우 이동 거리는 맨해튼 거리로 계산할 수 있다.
- 현재 위치가 바뀌는 시뮬레이션 문제에서는 매 단계마다 상태 갱신을 빼먹지 않아야 한다.
- 동률일 때만 `hand` 조건을 사용한다.

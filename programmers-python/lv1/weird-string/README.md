# 이상한 문자 만들기

## 문제 요약

문자열 `s`가 주어졌을 때, 각 단어의 짝수 번째 문자는 대문자로, 홀수 번째 문자는 소문자로 바꾼 문자열을 반환한다.
인덱스는 전체 문자열 기준이 아니라 각 단어 기준으로 계산한다.

문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/12930

## 접근 방식

처음에는 `split()`으로 단어를 나눈 뒤 각 단어를 변환했다.
하지만 `split()`은 연속된 공백과 앞뒤 공백 정보를 잃어버리기 때문에 정확성 테스트가 31.3점까지만 나왔다.

최종 풀이는 문자열을 한 글자씩 순회한다.
공백 보존이 핵심이므로 공백을 만나면 그대로 결과에 추가하고, 단어 안에서의 인덱스를 0으로 초기화한다.

## 핵심 코드

```python
def solution(s):
    answer = []
    index = 0

    for char in s:
        if char == " ":
            answer.append(char)
            index = 0
        else:
            answer.append(char.upper() if index % 2 == 0 else char.lower())
            index += 1

    return "".join(answer)
```

## 시간복잡도

O(n)

문자열을 한 번 순회한다.

## 공간복잡도

O(n)

결과 문자열을 만들기 위한 리스트가 필요하다.

## 배운 점

- `split()`은 공백을 기준으로 단어를 나눌 때 편하지만, 원래 공백 구조를 보존해야 하는 문제에는 조심해야 한다.
- 연속 공백, 앞 공백, 뒤 공백도 테스트 케이스로 직접 확인해야 한다.
- 문자열 변환 문제에서는 공백, 대소문자, 인덱스 기준이 어디인지 먼저 확인해야 한다.
- 문자열을 반복해서 `+=`로 붙이는 것보다 리스트에 담고 `join()`하는 방식이 더 안정적이다.

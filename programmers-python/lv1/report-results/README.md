# 신고 결과 받기

## 문제 요약

유저 목록 `id_list`, 신고 기록 `report`, 정지 기준 `k`가 주어진다.
각 유저가 신고한 대상 중 실제로 정지된 유저가 몇 명인지 계산해 메일 수를 반환한다.

문제 링크: [신고 결과 받기](https://school.programmers.co.kr/learn/courses/30/lessons/92334)

## 접근 방식

처음 풀이는 신고자별로 신고한 유저를 `set`에 저장했다.
이렇게 하면 같은 유저가 같은 유저를 여러 번 신고해도 한 번만 기록된다.

그다음 두 단계를 나눠서 처리했다.

1. 신고당한 유저별 신고 횟수를 계산한다.
2. 각 신고자가 신고한 유저 중 정지 기준 `k` 이상인 유저 수를 센다.

## 중복 신고 처리

이 문제에서 중요한 조건은 중복 신고다.

같은 유저가 같은 유저를 여러 번 신고해도 신고 횟수는 1번으로 처리해야 한다.
따라서 신고 관계를 저장할 때 `set`을 사용하거나, 처음부터 `set(report)`로 중복 신고를 제거하면 된다.

## 핵심 아이디어

- 신고자별 신고 대상: `dict[str, set[str]]`
- 신고당한 횟수: `dict[str, int]`
- 정지된 유저 판단: `reported_count[user] >= k`
- 최종 메일 수: `mail_count`, 내가 신고한 유저 중 정지된 유저의 수

## 시간복잡도

`O(n + r)`

`n`은 유저 수, `r`은 신고 기록 수다.
각 유저와 신고 기록을 몇 번씩 순회하므로 선형 시간에 처리할 수 있다.

## 공간복잡도

`O(n + r)`

유저별 신고 대상, 신고당한 횟수, 메일 수를 저장하기 위해 추가 공간을 사용한다.

## 배운 점

- 중복 제거가 필요하면 `set`을 먼저 떠올리자.
- 조회와 개수 계산이 함께 나오면 `dict`가 잘 맞는다.
- 문제를 한 번에 해결하려고 하기보다, 신고 관계 정리와 메일 수 계산을 단계로 나누면 더 읽기 쉽다.

# 로또의 최고 순위와 최저 순위

## 문제 요약

민우가 산 로또 번호 `lottos`와 당첨 번호 `win_nums`가 주어진다.
`lottos`에서 `0`은 알아볼 수 없는 번호다.

`0`이 모두 당첨 번호라고 가정했을 때의 최고 순위와, 모두 당첨 번호가 아니라고 가정했을 때의 최저 순위를 구한다.

## 테스트 15번 실패 원인

처음 풀이는 `rank`를 직접 줄여가며 계산했다.
하지만 `lottos`가 전부 `0`인 경우 최고 순위가 `0등`처럼 계산될 수 있었다.

```python
lottos = [0, 0, 0, 0, 0, 0]
win_nums = [38, 19, 20, 40, 15, 25]
```

이 경우 최고 순위는 6개를 모두 맞힐 수 있으므로 1등이고, 최저 순위는 하나도 못 맞힐 수 있으므로 6등이다.
정답은 `[1, 6]`이어야 한다.

## 접근 방식

현재 확실히 맞힌 개수와 알아볼 수 없는 번호의 개수를 따로 센다.

- `match_count`: 0이 아닌 번호 중 당첨 번호와 일치하는 개수
- `zero_count`: 알아볼 수 없는 번호의 개수
- 최고 순위 기준 맞힌 개수: `match_count + zero_count`
- 최저 순위 기준 맞힌 개수: `match_count`

## 순위 변환

맞힌 개수가 0개 또는 1개일 때는 모두 6등이다.
2개부터는 `7 - count`로 순위를 계산할 수 있다.

```python
def get_rank(count):
    return 6 if count < 2 else 7 - count
```

## 시간복잡도

`O(n)`

`n`은 `lottos`의 길이다.
문제에서는 길이가 항상 6이므로 상수 시간으로 볼 수도 있다.

## 공간복잡도

`O(1)`

개수 변수만 사용한다.
개선 풀이에서 사용하는 `set(win_nums)`도 번호 개수가 6개로 고정되어 있어 상수 공간으로 볼 수 있다.

## 배운 점

- 0개와 1개가 같은 6등으로 묶이는 조건을 따로 처리해야 한다.
- 순위 변환처럼 경계값이 있는 로직은 함수로 분리하면 실수를 줄일 수 있다.
- `0`처럼 미확정 값을 포함하는 문제는 최고/최저 경우를 따로 계산하자.

# H-Index

## 문제 요약

논문의 인용 횟수 목록 `citations`가 주어진다.
어떤 연구자의 H-Index는 `h`번 이상 인용된 논문이 `h`편 이상이고, 나머지 논문이 `h`번 이하 인용되었을 때의 최댓값이다.

문제 링크: [H-Index](https://school.programmers.co.kr/learn/courses/30/lessons/42747)

## 접근 방식

인용 횟수를 내림차순으로 정렬한다.
정렬된 배열에서 `i`번째 논문의 인용 횟수가 `i` 이상이면, 최소 `i`편의 논문이 `i`번 이상 인용된 상태라고 볼 수 있다.

그래서 `citation >= i`를 만족할 때마다 정답을 `i`로 갱신한다.

## 핵심 아이디어

- 인용 횟수가 큰 논문부터 본다.
- `enumerate(citations, start=1)`로 논문 수를 1부터 센다.
- `citation >= rank`이면 `rank`편 이상의 논문이 `rank`회 이상 인용된 것이다.
- 조건을 만족하는 가장 큰 `rank`가 H-Index다.

## 시간복잡도

`O(n log n)`

정렬에 `O(n log n)`이 걸리고, 이후 한 번 순회한다.

## 공간복잡도

기본 풀이는 `list.sort()`로 원본 리스트를 제자리 정렬하므로 추가 공간은 작다.
개선 풀이처럼 `sorted()`를 사용하면 정렬된 새 리스트를 만들기 때문에 `O(n)`의 추가 공간을 사용한다.

## 배운 점

H-Index는 정의를 코드 조건으로 바꾸는 것이 핵심이다.
내림차순 정렬 후 `citation >= rank`를 확인하면, `rank`편 이상의 논문이 `rank`회 이상 인용되었는지 자연스럽게 판단할 수 있다.

# 의상

## 문제 요약

스파이가 가진 의상 목록 `clothes`가 주어진다.
각 의상은 `[옷 이름, 종류]` 형태이고, 매일 최소 한 개 이상의 의상을 입어야 한다.
서로 다른 조합의 수를 반환한다.

문제 링크: [의상](https://school.programmers.co.kr/learn/courses/30/lessons/42578)

## 접근 방식

옷 이름 자체보다 종류별로 몇 개의 옷이 있는지가 중요하다.

각 종류에서는 아래 선택지가 있다.

- 해당 종류의 옷 중 하나를 입는다.
- 해당 종류의 옷을 입지 않는다.

따라서 종류별 선택지는 `옷 개수 + 1`개다.
모든 종류의 선택지를 곱한 뒤, 아무것도 입지 않는 경우 1개를 빼면 정답이 된다.

## 핵심 공식

종류별 옷 개수가 `a`, `b`, `c`라면:

```text
(a + 1) * (b + 1) * (c + 1) - 1
```

`+1`은 해당 종류를 입지 않는 경우를 뜻한다.
마지막 `-1`은 모든 종류를 입지 않는 경우를 제거하는 것이다.

## 시간복잡도

`O(n)`

의상 목록을 한 번 순회해서 종류별 개수를 세고, 종류 수만큼 다시 순회한다.

## 공간복잡도

`O(k)`

`k`는 의상 종류 수다.
개선 풀이에서는 종류별 개수만 저장한다.

## 배운 점

- 조합 문제처럼 보여도 종류별 개수를 세면 곱의 법칙으로 풀 수 있다.
- 각 종류에서 “선택하지 않는 경우”를 포함하려면 `count + 1`을 곱한다.
- 문제 조건이 “최소 한 개 이상 착용”이면 마지막에 아무것도 선택하지 않는 경우를 빼야 한다.
- 옷 이름이 실제 계산에 필요 없다면 리스트 대신 개수만 저장해도 된다.

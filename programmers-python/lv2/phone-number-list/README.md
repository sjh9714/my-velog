# 전화번호 목록

## 문제 요약

전화번호 목록 `phone_book`이 주어졌을 때, 어떤 번호가 다른 번호의 접두어인지 확인한다.
접두어 관계가 하나라도 있으면 `False`, 없으면 `True`를 반환한다.

문제 링크: [전화번호 목록](https://school.programmers.co.kr/learn/courses/30/lessons/42577)

## 실패 원인

처음 풀이는 `for i in phone_book`으로 순회하면서 같은 리스트에 `remove()`와 `append()`를 사용했다.
반복 중인 리스트를 수정하면 순회 순서가 꼬일 수 있다.

또 매번 나머지 전화번호 전체를 `tuple`로 만들고 `startswith()`를 호출했다.
이 구조는 모든 번호끼리 비교하는 방식에 가까워져 입력이 커지면 효율성 테스트에서 불리하다.

## 접근 방식

전화번호를 문자열 기준으로 정렬하면 접두어 관계가 있는 번호들은 서로 이웃한다.

예를 들어 `["1235", "12", "567"]`을 정렬하면 `["12", "1235", "567"]`이 된다.
이때 `"12"`가 `"1235"`의 접두어인지 바로 인접한 두 값만 확인하면 된다.

## 핵심 아이디어

- 전화번호 목록을 정렬한다.
- 인접한 두 번호만 비교한다.
- 뒤 번호가 앞 번호로 시작하면 접두어 관계이므로 `False`다.
- 끝까지 접두어 관계가 없으면 `True`다.

## 시간복잡도

`O(n log n)`

정렬에 `O(n log n)`이 걸리고, 인접한 번호 비교는 한 번의 순회로 처리한다.

## 공간복잡도

`O(n)`

`sorted(phone_book)`으로 정렬된 새 리스트를 만든다.

## 배운 점

- 반복 중인 리스트를 직접 수정하면 예상하지 못한 결과가 나올 수 있다.
- 접두어 문제는 정렬하면 비교 범위를 크게 줄일 수 있다.
- `a.startswith(b)`는 `a`가 `b`로 시작하는지 확인하는 함수이므로 방향을 조심해야 한다.

# 가장 큰 수

## 문제 요약

정수 배열 `numbers`가 주어졌을 때, 숫자를 이어 붙여 만들 수 있는 가장 큰 수를 문자열로 반환한다.

문제 링크: [가장 큰 수](https://school.programmers.co.kr/learn/courses/30/lessons/42746)

## 접근 방식

숫자 크기 자체를 기준으로 정렬하면 원하는 답을 만들 수 없다.
예를 들어 `3`과 `30`은 숫자로 보면 `30`이 더 크지만, 이어 붙이면 `"330"`이 `"303"`보다 크므로 `3`이 앞에 와야 한다.

따라서 숫자를 문자열로 바꾸고, 문자열을 반복한 값을 정렬 기준으로 사용한다.

## 핵심 아이디어

- 숫자를 문자열로 변환한다.
- `x * 3`을 정렬 key로 사용해 이어 붙였을 때 큰 순서를 만든다.
- 정렬된 문자열을 `"".join()`으로 합친다.
- 모든 숫자가 0이면 `"000"`이 아니라 `"0"`을 반환해야 한다.

## `x * 3`을 사용하는 이유

프로그래머스 조건에서 숫자는 `0` 이상 `1000` 이하이다.
서로 다른 자리수의 문자열을 비교해야 하므로, 각 문자열을 반복해서 비교 기준 길이를 맞춘다.

예를 들어 `3`과 `30`을 비교할 때는 아래처럼 볼 수 있다.

```text
"3" * 3  -> "333"
"30" * 3 -> "303030"
```

문자열 비교에서 `"333"`이 `"303030"`보다 앞서므로 `3`이 `30`보다 먼저 온다.
이 순서가 실제로 `"330"`을 만들어 `"303"`보다 큰 값을 만든다.

## `0` 조건을 문자열 상태에서 처리하는 이유

입력이 `[0, 0, 0]`이면 정렬 후 join 결과는 `"000"`이다.
하지만 정답은 `"0"`이어야 한다.

단순히 `return answer`만 하면 런타임 에러가 아니라 `"000"`을 반환하는 오답이 된다.
이 문제를 `str(int(answer))`로 해결하려고 하면 입력이 커졌을 때 매우 긴 문자열을 정수로 변환하면서 Python 정수 변환 제한이나 비용 때문에 런타임 에러가 날 수 있다.

그래서 `int(answer)`로 바꾸지 않고, 문자열 상태에서 첫 글자만 확인한다.

```python
return "0" if answer[0] == "0" else answer
```

정렬 결과의 첫 글자가 `"0"`이라는 것은 모든 숫자가 0이라는 뜻이다.

## 시간복잡도

`O(n log n)`

문자열 변환에 `O(n)`, 정렬에 `O(n log n)`이 걸린다.
정렬 key에서 문자열을 반복하지만 숫자 길이가 제한되어 있으므로 상수 비용으로 볼 수 있다.

## 공간복잡도

`O(n)`

숫자를 문자열로 바꾼 리스트와 최종 문자열을 저장한다.

## 배운 점

이어 붙여 가장 큰 수를 만드는 문제는 숫자 정렬이 아니라 문자열 정렬 문제로 봐야 한다.
자리수가 다른 숫자는 반복 문자열을 기준으로 비교하면 순서를 잡기 쉽다.
모든 값이 0인 경우는 `int()` 변환보다 문자열 첫 글자로 처리하는 편이 안전하다.

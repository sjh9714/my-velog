# 8주 코딩테스트 학습 진행표

## 목표

Python으로 프로그래머스 코딩테스트를 준비하면서 8주 동안 **문제 풀이 + 오답 복습 + 블로그 포스팅**을 함께 진행한다.

- 기준: 하루 1.5~2시간, 주 6일 문제 풀이, 일요일 복습/휴식
- 목표 문제 수: Lv.1 35~45문제, Lv.2 60~80문제, Lv.3 5~15문제
- 핵심 원칙: 많이 푸는 것보다 **틀린 문제를 다시 풀 수 있게 만드는 것**
- 문제 찾는 방법: 프로그래머스에서 아래 문제 제목을 검색하거나 공식 고득점 Kit에서 유형별로 선택
- 공식 고득점 Kit: https://school.programmers.co.kr/learn/challenges?tab=algorithm_practice_kit

## 문제 풀이 규칙

문제를 풀기 전에 아래 순서로 체크한다.

1. 입력 크기가 얼마인가?
2. O(n^2)이 가능한가?
3. 정렬하면 쉬워지는가?
4. dict, set, Counter를 쓰면 빨라지는가?
5. 큐, 스택, 힙이 필요한가?
6. 모든 경우를 봐도 되는가?
7. 그래프 탐색, DP, 이분탐색 문제인가?

## 유형별 추천 문제 은행

아래 문제는 하루 계획에서 부족할 때 추가로 풀기 좋다.

### 해시

- 완주하지 못한 선수
- 폰켓몬
- 전화번호 목록
- 의상
- 베스트앨범
- 할인 행사

### 스택/큐

- 같은 숫자는 싫어
- 올바른 괄호
- 기능개발
- 프로세스
- 다리를 지나는 트럭
- 주식가격
- 두 큐 합 같게 만들기
- 택배상자

### 힙

- 더 맵게
- 디스크 컨트롤러
- 이중우선순위큐

### 정렬

- K번째수
- 가장 큰 수
- H-Index
- 파일명 정렬
- 실패율

### 완전탐색

- 최소직사각형
- 모의고사
- 소수 찾기
- 카펫
- 피로도
- 모음사전
- 전력망을 둘로 나누기
- 후보키
- 수식 최대화

### DFS/BFS

- 타겟 넘버
- 게임 맵 최단거리
- 네트워크
- 단어 변환
- 여행경로
- 아이템 줍기
- 거리두기 확인하기
- 가장 먼 노드


## 블로그 포스팅 템플릿

```text
제목: [프로그래머스/Python] 문제명 풀이

1. 문제 요약
2. 입력 크기와 시간복잡도 판단
3. 처음 떠올린 접근
4. 실패한 부분 또는 반례
5. 최종 풀이 아이디어
6. Python 코드
7. 배운 점
8. 다음에 비슷한 문제를 만나면 떠올릴 신호
```

## 블로그 제목 예시

- [프로그래머스/Python] 완주하지 못한 선수 - Counter 풀이와 dict 풀이 비교
- [프로그래머스/Python] 기능개발 - 큐 없이도 풀 수 있는 배포 묶음 문제
- [프로그래머스/Python] 게임 맵 최단거리 - BFS가 필요한 이유
- [프로그래머스/Python] 입국심사 - 정답을 이분탐색하는 문제
- [프로그래머스/Python] 정수 삼각형 - dp[i][j] 의미 잡기

## 재풀이 규칙

틀린 문제는 아래 주기로 다시 푼다.

- 당일: 해설 보고 이해한 뒤 코드 다시 작성
- 2일 뒤: 아무것도 보지 않고 재풀이
- 1주 뒤: 제한시간 20~30분 안에 재풀이

## 실전 모의고사 규칙

- [ ] 제한시간 2시간
- [ ] 검색 금지
- [ ] 힌트 금지
- [ ] 쉬운 문제부터 풀기
- [ ] 20분 이상 막히면 다음 문제로 이동
- [ ] 끝난 뒤 풀이 순서, 막힌 이유, 반례를 블로그에 정리

## 최종 체크리스트

8주가 끝났을 때 아래 항목을 확인한다.

- [ ] Lv.1은 15~20분 안에 대부분 풀 수 있다.
- [ ] Lv.2 문제를 읽고 해시/스택/큐/정렬/탐색/DP 중 어떤 유형인지 판단할 수 있다.
- [ ] BFS 기본 코드를 외우지 않고도 작성할 수 있다.
- [ ] 이분탐색에서 condition(mid)를 직접 설계할 수 있다.
- [ ] 오답노트가 최소 30개 이상 쌓였다.
- [ ] 블로그 포스팅을 최소 40개 이상 작성했다.
- [ ] 자주 틀리는 유형 TOP 3를 알고 있다.
